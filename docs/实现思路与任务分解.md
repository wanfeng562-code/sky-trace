# Sky-Trace 实现思路与任务分解

> **当前状态（2026-05-18）**：M2～M3 能力已基本达成——三页前端、统一采集、回放、统计、GRD 网格、地名同步、SQLite 六表与性能优化已落地。详见 [系统前后端全面修复与优化.md](./系统前后端全面修复与优化.md)、[工程骨架说明.md](./工程骨架说明.md)。

## 1. 文档目标
本文件用于统一 Sky-Trace 的落地实现路径，明确前端、后端、Qt 桌面壳的职责边界，以及前后端联调交互点。

## 2. 目标范围（一个月内）
- 实现可演示的航班跟踪桌面平台。
- 两周内完成初版：可实时显示航班位置、轨迹、基础筛选与详情。
- 后两周持续改进：补充回放、统计、性能与 UI 优化。

## 3. 总体实现策略
采用分层并行开发：

- Qt 层（desktop）：提供桌面容器、窗口管理、后端进程管理。
- 前端层（client）：提供地图可视化、交互、状态管理、图表展示。
- 服务层（server）：提供数据采集、数据处理、接口服务、实时推送。

核心原则：
- 契约先行：后端先定义 API 契约，前端按契约开发。
- 最小闭环优先：先打通采集 -> 服务 -> 地图展示全链路。
- 迭代增强：初版可用后再扩展统计、回放、优化和 UI 打磨。

## 4. 功能拆分（按模块）

### 4.1 后端模块（你负责）
1. 数据采集模块
- 对接航班数据源（可先用模拟数据 + 真实数据双通道）
- 定时抓取与失败重试
- 数据去重与合法性校验（经纬度、速度、高度范围）

2. 数据处理模块
- 航班实体标准化（统一字段名、单位、时间格式）
- 轨迹拼接与历史窗口维护
- 异常数据过滤与日志记录

3. 接口服务模块
- REST API：航班列表、航班详情、区域筛选、历史轨迹
- WebSocket：实时增量推送
- OpenAPI 文档自动生成与维护

4. 存储与缓存模块
- SQLite 保存历史记录与回放片段
- 内存缓存维护在线航班快照

5. 运维与质量模块
- 配置管理（.env）
- 日志分级
- 基础单元测试与接口测试

### 4.2 前端模块（同学负责）
1. 地图展示模块
- 初始化地图底图与图层
- 绘制航班点、轨迹线、方向箭头
- 支持缩放级别和可见区域更新

2. 航班交互模块
- 航班列表、搜索、筛选（航班号、区域、状态）
- 选中航班高亮与详情卡片
- 自动刷新与断线重连提示

3. 实时通信模块
- WebSocket 连接与重连策略
- 增量更新本地状态（Pinia）
- 数据节流与防抖，避免频繁重渲染

4. 统计与回放模块（后期）
- ECharts 统计图
- 历史轨迹回放控制（开始、暂停、倍速）

5. UI 与工程模块
- 页面布局与样式体系
- 公共组件（面板、过滤器、状态标签）
- 异常态与空态设计

### 4.3 Qt 桌面模块（建议你主导）
1. 应用壳层
- 启动窗口、菜单、应用图标、日志面板入口

2. 前端嵌入
- 加载 Vue 构建产物（dist）
- 开发模式可指向本地 dev server

3. 后端管理
- 启动与监控 Python 服务进程
- 端口占用检测与健康检查
- 异常退出提示与自动重启策略

4. 本地桥接能力（可选）
- WebChannel 暴露配置读写、导出截图、打开日志目录等能力

## 5. 前后端联调交互点（重点）

### 5.1 数据模型契约（建议首批冻结）
FlightBrief（地图卡片）：
- flight_id
- callsign
- lat
- lon
- heading
- speed_kts
- altitude_ft
- updated_at

FlightDetail（详情面板）：
- 基础字段 + 起降机场、机型、状态、轨迹点列表

TrackPoint（轨迹点）：
- ts
- lat
- lon
- altitude_ft

### 5.2 API 契约（初版）
- GET /api/v1/flights
- GET /api/v1/flights/{flight_id}
- GET /api/v1/flights/{flight_id}/track
- GET /api/v1/health
- WS /api/v1/ws/flights

### 5.3 联调规则
- 后端字段变更必须先更新契约文档，再改代码。
- 前端不直接依赖临时字段，不猜字段语义。
- 每次联调固定验证清单：
  - 列表返回是否完整
  - WebSocket 推送是否连续
  - 单航班详情与轨迹是否匹配
  - 异常场景（断网、空数据）是否可恢复

## 6. 任务分解（可执行清单）

### 6.1 阶段 A：最小闭环（优先级 P0）
1. 后端：输出 flights 列表接口 + ws 推送
2. 前端：地图显示航班点 + 自动刷新
3. Qt：启动窗口加载前端页面
4. 联调：跑通采集 -> 接口 -> 地图展示

完成标准：
- 能看到实时点位刷新
- 无阻断级错误

### 6.2 阶段 B：核心体验（优先级 P1）
1. 后端：详情接口、轨迹接口、筛选参数
2. 前端：筛选、搜索、详情卡片、轨迹高亮
3. Qt：后端进程守护与健康检测
4. 联调：稳定性测试（长时间运行）

完成标准：
- 能按航班查看详情和轨迹
- 断线能重连，服务异常有提示

### 6.3 阶段 C：增强能力（优先级 P2）
1. 后端：历史回放数据查询、统计接口
2. 前端：回放控制、统计图表
3. Qt：日志导出、设置页（可选）
4. 联调：演示脚本与录屏场景打磨

完成标准：
- 有可展示的回放与统计能力
- UI 观感和交互流畅度明显提升

## 7. 双人协作分工建议

你（后端负责人）：

共同负责：
- 联调验证

- 演示版本打包
