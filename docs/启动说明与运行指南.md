# Sky-Trace 启动说明与运行指南

## 1. 适用范围
本说明适用于当前 Sky-Trace 工程骨架，目标是帮助开发阶段快速启动后端、前端和桌面壳并完成最小联调。

## 2. 当前可启动项

- server（FastAPI 后端）
- client（Vue3 前端开发服务器）
- desktop（PySide6 桌面壳）

---

## 3. 启动前检查

```powershell
python --version    # 需要 3.10+
node --version      # 需要 20 LTS+
npm.cmd --version
```

---

## 4. 后端启动（server）

### 4.1 首次启动（含 API Key 填写）

```powershell
cd server

# 创建虚拟环境
python -m venv .venv
.\.venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 从模板复制配置文件
copy .env.development.example .env
```

用编辑器打开 `server/.env`，填写以下四个占位符（其余字段保持默认即可）：

| 字段                  | 如何获取                                                         |
| --------------------- | ---------------------------------------------------------------- |
| `OPENSKY_USERNAME`    | 在 https://opensky-network.org/ 注册账号，用户名即登录名         |
| `OPENSKY_PASSWORD`    | 同上，账号密码                                                   |
| `OPENWEATHER_API_KEY` | 在 https://openweathermap.org/api 注册，进入 "API keys" 页面复制 |
| `AIRLABS_API_KEY`     | 在 https://airlabs.co/ 注册，进入 Dashboard 复制 API Key         |

**可选：国内网络代理**

若直连 `opensky-network.org` / `openweathermap.org` 超时（常见于国内网络环境），在 `server/.env` 末尾加一行：

```ini
HTTP_PROXY=http://127.0.0.1:7890
```

将 `7890` 替换为本机代理端口。不需要代理时留空或删除该行。

> ⚠️ `.env` 已在 `.gitignore` 中排除，**不会被提交到仓库**。  
> 请勿将真实 Key 填写到 `.env.development.example` 或其他可提交文件中。

填写完成后启动服务：

```powershell
# 必须在 server/ 目录下，或设置 PYTHONPATH
cd D:\Gitee\Sky-Trace\server
或 $env:PYTHONPATH = "D:\Gitee\Sky-Trace\server"

python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

### 4.2 各字段不填写时的影响

| 字段                        | 不填写的影响                                          |
| --------------------------- | ----------------------------------------------------- |
| `OPENSKY_USERNAME/PASSWORD` | 匿名访问，额度降至 400 点/日（全球模式约 1 小时耗尽） |
| `OPENWEATHER_API_KEY`       | 天气/空气质量缓存为空，前端无天气数据                 |
| `AIRLABS_API_KEY`           | 航班无航班号、起降机场、机型等富集字段                |

### 4.3 验证后端是否正常

在浏览器访问：

| 地址                                                                        | 预期结果                                                          |
| --------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| `http://127.0.0.1:8000/api/v1/health`                                       | `{"status":"ok","db":"ok"}`                                       |
| `http://127.0.0.1:8000/api/v1/flights`                                      | 含 `total` 与 `items` 的航班列表                                  |
| `http://127.0.0.1:8000/api/v1/flights/summary/stats`                        | 含 `total/on_ground/airborne/by_category/by_altitude_band` 的统计 |
| `http://127.0.0.1:8000/api/v1/datahub/status`                               | 三层采集状态，`last_count` 有值                                   |
| `http://127.0.0.1:8000/api/v1/datahub/weather`                              | IATA 键字典，如 `{"CAN":{...},"HKG":{...}}`                       |
| `http://127.0.0.1:8000/api/v1/datahub/weather/nearest?lat=23.39&lon=113.30` | 最近有缓存数据的枢纽天气，`weather` 字段非 null                   |
| `http://127.0.0.1:8000/api/v1/datahub/air_quality`                          | 各枢纽 AQI + 污染物浓度                                           |
| `http://127.0.0.1:8000/api/v1/datahub/quota`                                | 各 API 今日调用次数与预算                                         |
| `http://127.0.0.1:8000/api/v1/playback?start=<iso>&end=<iso>&interval=60`   | 历史回放帧列表（服务启动约 5 分钟后有数据）                       |
| `http://127.0.0.1:8000/debug/flights-dashboard`                             | 调试可视化页面                                                    |

### 4.4 OpenSky 轮询策略（当前）

开发档位默认**关闭**活跃时间窗口，全天匀速 90 秒轮询：

- 约 960 次/日 × 4 点 = **3840 点/日**（预算上限 4000 点/日，余量充足）
- 全球模式（`OPENSKY_BBOX` 留空）每次约 6000 架飞机

若需开启活跃窗口（低频/高频混合），在 `.env` 中设置：
```ini
DEV_REALTIME_USE_ACTIVE_WINDOW=true
DEV_REALTIME_ACTIVE_INTERVAL_SECONDS=15   # 活跃时段间隔
DEV_REALTIME_IDLE_INTERVAL_SECONDS=90     # 空闲时段间隔
DEV_REALTIME_ACTIVE_START_HOUR=8          # 活跃窗口起始小时
DEV_REALTIME_DAILY_BUDGET_CALLS=1000      # 预算上限（软截断）
```

### 4.5 关闭后端服务

```powershell
# 方式 1：在运行 uvicorn 的终端按 Ctrl+C（推荐）

# 方式 2：按端口强制结束（将下面的 7332 替换为 netstat 查到的实际 PID）
netstat -ano | findstr :8000
taskkill /F /PID 7332
```

```powershell
cd client

```

## 6. 后端日志说明

日志格式：`时间 | 级别 | 模块 | 消息`

常见日志及含义：

| 日志                                                                   | 含义                                             |
| ---------------------------------------------------------------------- | ------------------------------------------------ |
| `unified pipeline started ... realtime=90s env=300s commercial=86400s` | 服务正常启动                                     |
| `AirLabs bulk snapshot: N flights enriched`                            | 商业层采集成功，N 条富集                         |
| `environment collected: N/20 hubs OK`                                  | 环境层完成，N 个枢纽成功（<20 属正常，网络波动） |
| `Weather fetch failed for XXX: [TimeoutError]`                         | 单枢纽超时，已跳过，不影响其他枢纽               |
| `Retrying N failed hub(s) after 5 s: [...]`                            | 首次失败后自动重试，大多数枢纽重试后恢复         |
| `environment collection failed [TimeoutError]`                         | 整轮环境层失败，下轮自动重试                     |
| `realtime collection failed [...]`                                     | OpenSky 失败，已回退 mock 数据                   |

## 7. PYTHONPATH 说明

uvicorn 必须在 `server/` 目录下或显式设置 `PYTHONPATH` 才能正确导入模块：

```powershell
# 方式 A（推荐）：进入 server 目录
cd D:\Gitee\Sky-Trace\server
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

# 方式 B：从任意目录启动，手动设置 PYTHONPATH
$env:PYTHONPATH = "D:\Gitee\Sky-Trace\server"
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

`.env` 文件路径解析已固定为相对 `server/` 目录，无论从哪里启动均可正确加载。

## 5. 前端启动（client）

### 5.1 首次启动
```powershell
cd client
npm.cmd install
npm.cmd run dev
```

### 5.2 验证前端是否正常
浏览器打开：

- `http://localhost:5173`

预期：
- 页面可打开，左侧地图显示航班点位，右侧列表面板展示航班条目
- 顶部工具栏显示实时航班总数
- 搜索框可按呼号/航班号过滤，Tab 可切换全部/飞行中/地面
- 点击地图点或列表条目，左下角弹出详情卡片（含高度、速度、起降机场、机型、天气）
- 若后端断开，列表顶部显示「⚠ 实时连接已断开，正在重连...」横幅
- WebSocket 重连策略：2s → 4s → 8s → 15s → 30s 退避

## 6. 桌面壳启动（desktop）

### 6.1 首次启动
```powershell
cd desktop
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

### 6.2 验证桌面壳是否正常
预期：
- 弹出 Sky-Trace 窗口
- 若 `client/dist/index.html` 不存在，则自动尝试加载 `http://localhost:5173`
- 桌面壳会尝试拉起后端进程

## 7. 推荐联调顺序（P0）
按以下顺序最稳定：

1. 先启动后端（确认 health/flights 可访问）
2. 再启动前端（确认页面可连通后端）
3. 最后启动桌面壳（确认窗口能加载前端）

## 8. 常见问题与处理

### 8.1 `npm --version` 无输出
现象：
- PowerShell 中 `npm --version` 没有输出

处理：
- 使用 `npm.cmd --version`、`npm.cmd install`、`npm.cmd run dev`

### 8.2 端口冲突
现象：
- 8000 或 5173 端口被占用

处理：
- 修改后端启动端口并同步更新 `client/.env.development` 中的 `VITE_API_BASE_URL` 与 `VITE_WS_URL`
- 或停止占用 8000 端口的旧进程后再启动（参考第 4.4 节）

### 8.3 前端页面打开但无数据
排查顺序：

1. 检查后端是否已启动
2. 访问 `http://127.0.0.1:8000/api/v1/flights` 是否有返回
3. 检查浏览器控制台是否出现网络错误
4. 访问 `http://127.0.0.1:8000/api/v1/datahub/status` 查看三层采集状态

### 8.4 desktop 打开白屏
排查顺序：

1. 确认前端开发服务器正在运行（`http://localhost:5173`）
2. 确认 `desktop` 环境已安装 PySide6
3. 确认后端已启动或可由 desktop 拉起

### 8.5 出现 `/hybridaction/... 404`
现象：
- 日志出现 `/hybridaction/...`、`/.well-known/...` 等 404

处理：
- 多数为浏览器插件或本机程序探测请求，不是项目接口，可忽略。
