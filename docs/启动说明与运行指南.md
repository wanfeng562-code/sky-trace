# Sky-Trace 启动说明与运行指南

## 1. 适用范围
本说明适用于当前 Sky-Trace 工程骨架，目标是帮助开发阶段快速启动后端、前端和桌面壳并完成最小联调。

## 2. 当前可启动项
目前可启动的模块如下：

- server（FastAPI 后端）
- client（Vue3 前端开发服务器）
- desktop（PySide6 桌面壳）

说明：
- `scripts/dev-start.ps1` 当前是命令提示脚本（仅输出命令），不是一键拉起脚本。

## 3. 启动前检查
建议先确认本机环境：

- Python 3.10+
- Node.js 20+
- npm 可用（PowerShell 下建议优先使用 `npm.cmd`）

检查命令示例：

```powershell
python --version
node --version
npm.cmd --version
```

## 4. 后端启动（server）

### 4.1 首次启动
```powershell
cd server
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
copy .env.development.example .env
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

说明：
- 默认使用 `APP_PROFILE=development`（开发配置）。
- 如需切换发行配置，可在 `.env` 中设置 `APP_PROFILE=release`。

### 4.2 验证后端是否正常
在浏览器访问：

- `http://127.0.0.1:8000/api/v1/health`
- `http://127.0.0.1:8000/api/v1/flights`
- `http://127.0.0.1:8000/api/v1/datahub/status`
- `http://127.0.0.1:8000/api/v1/datahub/snapshot`
- `http://127.0.0.1:8000/debug/flights-dashboard`

预期：
- health 返回 `status: ok`
- flights 返回 `total` 与 `items`
- datahub/status 返回 `profile/realtime/environment/commercial` 状态
- 访问 `http://127.0.0.1:8000/` 会自动跳转到 dashboard

### 4.3 开发档位 OpenSky 轮询策略
开发档位默认启用时段化轮询：

- 活跃时段：15 秒
- 非活跃时段：40 秒
- 活跃时段起点：08:00
- 按 4000 次/日预算自动计算活跃窗口，当前结束时间约为 20:16

## 5. 前端启动（client）

### 5.1 首次启动
```powershell
cd client
npm.cmd install
npm.cmd run dev
```

### 5.2 验证前端是否正常
浏览器打开：

- `http://localhost:5173`

预期：
- 页面可打开
- 顶部显示航班数量
- 列表面板可看到航班条目（若后端已启动）

## 6. 桌面壳启动（desktop）

### 6.1 首次启动
```powershell
cd desktop
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

### 6.2 验证桌面壳是否正常
预期：
- 弹出 Sky-Trace 窗口
- 若 `client/dist/index.html` 不存在，则自动尝试加载 `http://localhost:5173`
- 桌面壳会尝试拉起后端进程

## 7. 推荐联调顺序（P0）
按以下顺序最稳定：

1. 先启动后端（确认 health/flights 可访问）
2. 再启动前端（确认页面可连通后端）
3. 最后启动桌面壳（确认窗口能加载前端）

## 8. 常见问题与处理

### 8.1 `npm --version` 无输出
现象：
- PowerShell 中 `npm --version` 没有输出

处理：
- 使用 `npm.cmd --version`、`npm.cmd install`、`npm.cmd run dev`

### 8.2 端口冲突
现象：
- 8000 或 5173 端口被占用

处理：
- 修改后端启动端口并同步更新 `client/.env.development` 中的 `VITE_API_BASE_URL` 与 `VITE_WS_URL`
- 或停止占用 8000 端口的旧进程后再启动

### 8.3 前端页面打开但无数据
排查顺序：

1. 检查后端是否已启动
2. 访问 `http://127.0.0.1:8000/api/v1/flights` 是否有返回
3. 检查浏览器控制台是否出现网络错误
4. 访问 `http://127.0.0.1:8000/api/v1/datahub/status` 查看三层采集状态

### 8.4 desktop 打开白屏
排查顺序：

1. 确认前端开发服务器正在运行（`http://localhost:5173`）
2. 确认 `desktop` 环境已安装 PySide6
3. 确认后端已启动或可由 desktop 拉起

### 8.5 出现 `/hybridaction/... 404`
现象：
- 日志出现 `/hybridaction/...`、`/.well-known/...` 等 404

处理：
- 多数为浏览器插件或本机程序探测请求，不是项目接口，可忽略。

## 9. 相关文件
- 后端入口：`server/app/main.py`
- 前端入口：`client/src/main.ts`
- 桌面入口：`desktop/main.py`
- 启动脚本（提示版）：`scripts/dev-start.ps1`
