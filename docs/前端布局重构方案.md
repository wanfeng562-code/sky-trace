# Sky-Trace 前端布局重构方案

> **实施状态（2026-05-18）**：阶段 1 已部分落地——`AppShell.vue`、`components/map/*`、`components/flight/*`、`styles/tokens.css`、顶栏三路由与语言切换。阶段 2（浅色主题、右栏可折叠等）见下文待办。  
> 当前页面行为以 [前端页面说明文档.md](前端页面说明文档.md) 为准；地图视觉见 [地图可视化设计说明.md](地图可视化设计说明.md)。

## 1. 背景与目标

### 1.1 用户诉求

| 诉求 | 说明 |
|------|------|
| 观感舒适 | 长时间盯看不产生视觉疲劳，避免「赛博朋克 / 军用雷达」过重 |
| Release 质感 | 接近现代 SaaS / Flightradar24 类 Web App，而非毕设 Demo |
| 功能整合 | 减少「一功能一悬浮钮」，用统一导航与面板收纳 |
| 地图优先 | 未操作时尽量把视野还给地图 |

### 1.2 设计原则（与 Gemini 建议对齐）

1. **做减法**：中性灰底、细边框、轻阴影；主色仅用于选中态与关键操作。
2. **App Shell 优先**：全局顶栏 + 地图页左右分栏，控件有固定「住址」。
3. **渐进披露**：列表常显、高级筛选与图层设置按需展开；详情右栏无选中时可收起。
4. **一致性**：三路由（地图 / 统计 / 回放）共用同一套 Token、顶栏高度与面板样式。
5. **地图与 UI 分离**：UI 可浅色/可切换主题；地图仍可用深色底图 + 琥珀金飞机（已在地图文档定稿）。

## 2. 现状诊断（As-Is）

### 2.1 当前信息架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│ App.vue 全局导航 (48px)  Logo | 地图·统计·回放 | WS 实时/断线              │
├─────────────────────────────────────────────────────────────────────────┤
│ MapView 任务栏 (44px)  [图层][AQI] | 追踪/飞行中/地面统计 | [侧栏开关]    │  ← 与顶栏功能重叠
├──────────┬──────────────────────────────────────────────┬───────────────┤
│ 图层抽屉  │              MapLibre 地图                    │ 右侧上下文面板   │
│ (236px)  │  · 右上：缩放/指南针/定位 (MapLibre 默认)       │ (320px 常显)   │
│ 按需滑出  │  · 右下：比例尺                              │ Tab:          │
│          │                                              │ 航班|枢纽|详情|时刻│
│          │                                              │ · 搜索+筛选+列表 │
│          │                                              │   全挤在右侧     │
└──────────┴──────────────────────────────────────────────┴───────────────┘
```

**统计页**（`/stats`）：全屏 KPI + ECharts，无侧栏，风格与地图页面板组件未完全统一。  
**回放页**（`/playback`）：独立顶栏查询区 + 全屏地图 + 左上毛玻璃统计卡 + 底部播放条，控件分布与主地图页不一致。

### 2.2 主要问题清单

| # | 问题 | 现状位置 | 影响 |
|---|------|----------|------|
| P1 | **双顶栏** | `App.vue` + `MapView.task-bar` | 垂直空间浪费；图层/AQI 与全局导航割裂 |
| P2 | **右栏过重且常显** | `context-panel` 固定 320px | 地图有效宽度偏小；筛选展开时列表可视区更窄 |
| P3 | **图层入口分散** | 任务栏「图层」「AQI」+ 抽屉内重复开关 | 用户不知去哪改设置；AQI 与图层抽屉逻辑重复 |
| P4 | **列表与筛选同栏堆叠** | `FlightListPanel` 内搜索 + Pill + 可折叠高级筛选 + 虚拟列表 | 高级筛选占高过大，列表滚动体验差 |
| P5 | **无「地图工具」统一容器** | MapLibre 默认控件 + 自定义 Geolocate | 右上多个独立方块，与整体 UI 语言不统一 |
| P6 | **视觉偏重** | 深色 Slate 全站、`uppercase` 小节标题、WS 绿点 glow、`backdrop-filter` | 「监控系统」感强，不够轻 |
| P7 | **三页布局不统一** | Stats / Playback 各自一套顶栏与浮层 | 切换路由时有「三个产品拼在一起」的感觉 |

### 2.3 已有优点（重构时保留）

- 全局路由与 Pinia 状态已清晰，`App.vue` 具备 App Shell 雏形。
- 右侧 **Tab 上下文**（航班 / 枢纽 / 详情 / 时刻）信息架构合理，选中航班自动切详情已实现。
- 图层抽屉内容完整（底图来源、样式、叠加层、标注语言）。
- `FlightListPanel` 高级筛选能力完备（高度/速度/类型/机型/四级地区）。
- Design Token 已在 `App.vue :root` 定义，便于一次性升级主题。

## 3. 目标架构（To-Be）

### 3.1 整体 App Shell

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 全局顶栏 (56px) — 唯一顶栏，地图页不再叠加 task-bar                      │
│ ┌─────────────┬──────────────────────────────┬────────────────────────┐ │
│ │ Logo+名称   │  地图 | 统计 | 回放 (Segment)  │ 航班数·WS·刷新·主题(可选)│ │
│ └─────────────┴──────────────────────────────┴────────────────────────┘ │
├──────────┬──────────────────────────────────────────────┬─────────────┤
│ 左侧栏    │                 地图主舞台                     │ 右侧栏       │
│ (可折叠)  │                                              │ (按需显隐)   │
│ 360px    │  右下：地图控件组 (缩放/指南针/定位)             │ 360px       │
│          │  右下上：图层 FAB → Popover                    │             │
│ Tab:     │                                              │ Tab:        │
│ · 航班   │                                              │ · 详情       │
│ · 筛选   │                                              │ · 时刻       │
│          │                                              │ (无选中时可关) │
└──────────┴──────────────────────────────────────────────┴─────────────┘
```

> **与 Gemini 差异说明**：Gemini 建议「左侧列表 + 右侧详情」。本方案采用 **左：列表+筛选 / 右：详情+时刻**，枢纽列表仍放左侧「航班」同级 Tab（或左侧第三 Tab「枢纽」），避免详情与长列表抢同一列高度。若团队更倾向「右栏全能」，可退化为单列右栏（见 §6.2 备选）。

### 3.2 顶栏（`App.vue` 增强）

| 区域 | 内容 | 从现状迁移 |
|------|------|------------|
| 左 | Sky-Trace Logo + 名称 | 已有 `nav-brand` |
| 中 | 路由 Segment：`实时地图` / `数据统计` / `历史回放` | 已有 `nav-links`，文案可更产品化 |
| 右 | ① 简况：追踪数 / 飞行中（仅地图路由或全局） ② WS 状态 ③ 上次快照时间（可选） ④ 主题切换（可选 Phase 2） | 自 `MapView.task-bar` **上移** |

**删除**：`MapView` 内整条 `task-bar`（§4.1）。

### 3.3 左侧栏（新建 `MapSidebar.vue`）

| Tab | 组件 | 说明 |
|-----|------|------|
| **航班** | `FlightListPanel`（精简版） | 仅保留：搜索框、状态 Pill（全部/空中/地面）、结果计数、虚拟列表 |
| **筛选** | `FlightFilterPanel`（新建，从列表拆出） | 高度/速度/类型/机型/四级地区/范围；支持「已启用 N 项」摘要 |
| **枢纽** | `HubAirportListPanel` | 自右栏迁入，与列表同级 |

- 默认宽度 **360px**，可折叠为 **48px** 图标条（仅显示 Tab 图标）。
- 折叠状态写入 `localStorage`（`map.sidebar.collapsed`）。
- 移动端（&lt;768px）：左侧栏改为 **底部 Sheet** 或全屏抽屉。

### 3.4 右侧栏（`MapContextPanel.vue` 重构）

| 状态 | 行为 |
|------|------|
| 无选中航班、无时刻表机场 | **默认隐藏**（宽度 0），地图全宽 |
| 选中航班 | 滑出 **详情** Tab，`FlightDetailCard` |
| 点击枢纽「时刻表」 | 滑出 **时刻** Tab，`SchedulePanel` |
| 用户手动固定 | 提供图钉按钮「保持打开」 |

- 宽度 **360px**，与左侧对称。
- **移除**右侧「航班」「枢纽」Tab（已迁左）。
- 保留 `panelCollapsed` 逻辑，默认 `true`（未选中即折叠）。

### 3.5 地图浮动控件

| 控件组 | 位置 | 形态 |
|--------|------|------|
| **Navigation** | 地图右下 | 单卡片：`+` / `−` / 指南针，圆角 12px，实色底 `--bg-elevated`，1px 边框 |
| **Geolocate** | 紧挨导航组上方 | 与导航组同一视觉容器或相邻胶囊 |
| **Layers** | 导航组上方 | 单按钮「图层」→ Popover（内容 = 现 `layer-drawer` 全部项） |
| **Scale** | 左下 | 保持 MapLibre ScaleControl，样式与 Token 对齐 |

**删除**：左侧 `layer-drawer` 滑出层；任务栏「图层」「AQI」独立按钮。

Popover 结构建议：

```
图层
├── 底图来源 (MapTiler / Stadia / OpenFreeMap)
├── 底图样式 (网格按钮)
├── 叠加层 (Switch 列表)
│   ├── AQI / PM2.5 / 风温 / 密度 / 枢纽 / 天气网格
└── 标注语言 (行数 + 语言选择)
```

### 3.6 统计页与回放页

| 页面 | 调整 |
|------|------|
| **统计** | 内容区 `max-width: 1280px` 居中；KPI 卡与图表使用与地图侧栏相同的 `--surface` / `--radius-lg`；顶栏仅保留全局导航 |
| **回放** | 查询条收入顶栏下方 **单行工具带**（高度 48px），与地图页工具带组件复用 `PlaybackToolbar.vue`；去掉左上毛玻璃浮卡，统计数字并入工具带或播放条 |

## 4. 视觉与设计 Token 升级

### 4.1 色彩（建议）

在 `App.vue` 或 `styles/tokens.css` 扩展 **双主题**：

| Token | 暗色（默认，地图友好） | 浅色（可选） |
|-------|------------------------|--------------|
| `--bg-base` | `#0f1419` | `#f8fafc` |
| `--bg-surface` | `#1a2332` | `#ffffff` |
| `--bg-elevated` | `#243044` | `#ffffff` |
| `--text-primary` | `#e8edf4` | `#0f172a` |
| `--text-muted` | `#7a8ba3` | `#64748b` |
| `--accent` | `#3b82f6` | `#2563eb` |
| `--border` | `rgba(255,255,255,0.08)` | `#e2e8f0` |

- **去掉**：WS 绿点 `box-shadow: 0 0 6px`、大面积 `backdrop-filter`（仅保留 Toast 等极少数场景）。
- **阴影**：`--shadow-panel: 0 4px 24px rgba(0,0,0,0.12)` 单层，不用 `--shadow-lg` 堆叠。

### 4.2 排版与间距

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| L1 | 15–16px | 600 | 航班呼号、面板标题 |
| L2 | 13px | 500 | 正文、按钮 |
| L3 | 11px | 400 | 辅助信息（高度/速度/机场） |

- 间距统一 **4px 网格**：`4 / 8 / 12 / 16 / 24`。
- 小节标题：**取消全局 `text-transform: uppercase`**，改用 12px + `--text-muted` 即可。

### 4.3 组件样式约定

| 组件 | 规范 |
|------|------|
| 按钮 | 高度 32px，圆角 8px；Primary 仅一个/屏 |
| 输入框 | 高度 36px，focus 环 `--accent` 1px |
| Tab | Segment 式（顶栏路由）或下划线式（侧栏 Tab），不混用 |
| 列表项 | 行高 56px，hover `--bg-elevated`，选中左边框 3px `--accent` |

## 8. 备选方案与决策点

### 8.1 单列右栏（Gemini 原版）

**做法**：保留现有右栏 320px，内 Tab：`航班 | 筛选 | 枢纽 | 详情 | 时刻`。  
**优点**：改动最小，不新建左栏。  
**缺点**：筛选+列表+枢纽仍挤一侧，地图宽度损失大。  
**建议**：仅当工期极紧时采用；否则采用 §3.3 左列表右详情。

### 8.2 浅色默认 vs 深色默认

| 选项 | 说明 |
|------|------|
| A. UI 深色 + 地图深色底图 | 现状延续，改动小 |
| B. UI 浅色 + 地图深色底图 | 更接近 FR24；侧栏白、地图黑，对比清晰 **（推荐）** |
| C. 全站浅色含地图浅色底图 | 飞机琥珀金需重新调描边对比度 |

### 8.3 枢纽列表位置

| 选项 | 说明 |
|------|------|
| 左栏第三 Tab | 与航班/筛选并列 **（推荐）** |
| 顶栏下拉「工具」 | 适合枢纽较少的场景 |
| 地图点击仅时刻表 | 列表入口弱，不推荐 |

## 9. 风险与回归清单

| 风险 | 缓解 |
|------|------|
| 地图 `resize` 时机 | 侧栏动画 `transitionend` 后 `map.resize()`（现有逻辑保留） |
| Popover 被地图吞事件 | Popover 挂到 `body` 或 `map-shell` 外一层 |
| 筛选 Tab 与列表 Tab 分离后状态不同步 | 共用 `useFlightStore`，`filteredFlights` 不变 |
| 右栏默认隐藏后用户找不到详情 | 点飞机仍自动展开；首次使用可 Toast 提示一次 |

**回归测试**：

## 11. 总结

| 维度 | 现状 | 目标 |
|------|------|------|
| 顶栏 | 双层（48+44px） | 单层 56px，路由+状态合一 |
| 航班/筛选 | 全在右栏 320px | 左栏 360px，Tab 分离 |
| 详情/时刻 | 右栏 Tab 之一 | 右栏按需滑出，默认隐藏 |
| 图层 | 左抽屉 + 任务栏重复 | 右下 Popover 唯一入口 |
| 地图控件 | MapLibre 默认样式 | 统一卡片/胶囊 |
| 气质 | 深色控制台 | 现代 SaaS，轻边框轻阴影 |

按 **Phase 1 → 4** 推进即可达到可对外演示的 Release 布局；Phase 5–6 用于三页统一与响应式。实施时以**组件拆分 + Token 先行**为主，避免在 `MapView.vue` 单文件内继续堆叠样式。
