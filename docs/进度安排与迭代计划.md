# Sky-Trace 进度安排与迭代计划（4 周）

## 1. 计划目标
- 总周期：4 周
- 前 2 周：完成初版开发（可演示）
- 后 2 周：持续改进（功能补充、性能优化、UI 提升）

## 2. 里程碑定义
- M1（第 1 周末）：最小链路跑通
- M2（第 2 周末）：初版可演示版本完成
- M3（第 3 周末）：增强功能完成并进入优化
- M4（第 4 周末）：终版交付（文档、演示、答辩材料）

## 3. 周计划（详细）

### 第 1 周：基础搭建与最小闭环（M1）
目标：跑通 "采集 -> 后端 -> 前端地图 -> Qt 壳"。

后端（负责人）：
- 完成项目基础结构与依赖管理
- 建立航班数据模型与基础采集流程
- 提供 GET /api/v1/flights 与 /health
- 提供 WS /api/v1/ws/flights（基础推送）

前端（同学）：
- 初始化 Vue3 + Vite + TypeScript 工程
- 接入地图并渲染航班点
- 完成 WebSocket 接收与基础状态管理

Qt（负责人）：
- 建立 PySide6 壳层工程
- 加载前端页面（开发模式可指向 dev server）

本周验收：
- 能看到地图点位实时变化
- 系统可连续运行 30 分钟无阻断问题

### 第 2 周：初版功能完善（M2）
目标：形成可演示初版。

后端（负责人）：
- 完成详情接口与轨迹接口
- 支持筛选参数（区域、航班号）
- 完成 SQLite 历史落盘与基础日志

前端（同学）：
- 完成航班列表、搜索、筛选
- 完成航班详情卡片与轨迹高亮
- 处理断线重连、空态与错误提示

Qt（负责人）：
- 加入后端进程健康检测与异常提示

本周验收：
- 可演示完整流程：查找航班 -> 查看详情 -> 查看轨迹
- 初版问题清单（Bug List）建立并排序

### 第 3 周：增强与优化（M3）
目标：增强功能，提升可用性与观感。

后端（负责人）：
- 增加历史回放查询接口
- 增加统计接口（区域数量、速度分布等）
- 优化数据处理与缓存策略

前端（同学）：
- 增加回放控制组件
- 增加 ECharts 统计视图
- 优化地图渲染性能和交互反馈

Qt（负责人）：
- 增加日志入口、配置入口（可选）

本周验收：
- 回放与统计功能可用
- 高负载场景下页面仍保持可交互

### 第 4 周：打磨与交付（M4）
目标：形成稳定终版与完整文档。

后端（负责人）：
- 修复高优先级问题
- 补充接口文档与测试说明

前端（同学）：
- UI 细节统一（间距、颜色、层级、提示）
- 完成演示路径优化与异常态完善

共同任务：
- 全量联调回归
- 输出最终演示脚本
- 完成课程报告中的技术与进度章节

本周验收：
- 发布终版 tag（例如 v1.0.0-course）
- 具备稳定演示能力

## 4. 每周固定节奏
- 周一：确认本周目标与任务拆分
- 周三：中期检查（风险、阻塞、变更）
- 周五：集成验证与问题归档
- 周末：机动修复与文档更新

## 5. 日常协作节奏（建议）
- 每日 10 分钟站会：昨天完成、今天计划、阻塞项
- 每晚一次 develop 集成与冒烟验证
- 每次合并前检查：可运行、可构建、文档同步

## 6. 验收标准（按阶段）
- M1：可实时显示航班点
- M2：可筛选、可查看详情和轨迹
- M3：可回放、可统计、性能可接受
- M4：终版稳定、文档完整、演示顺畅

## 7. 风险缓冲安排
- 每周预留 0.5 天应对突发问题
- 第 4 周前半段仅做修复和优化，避免新增高风险需求
- 若外部数据源异常，切换本地模拟数据保障演示

## 8. 产出物清单
- 代码：client、server、desktop 三部分
- 文档：技术栈、协作规范、实现思路、进度计划、接口文档、测试记录
- 演示：演示脚本、关键截图、必要录屏
