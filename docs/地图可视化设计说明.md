# Sky-Trace 地图可视化设计说明

本文档整理主地图（`MapView`）与回放页（`PlaybackView`）在底图、飞机、枢纽、天气网格与面板上的视觉与交互约定。  
> 最后更新：**2026-05-18**。设计参考 [FlightAware](https://www.flightaware.com/news/article/Map-Improvements--View-All-Planes-Nearby-Airports-Premium-Map-Layers/197) 与 [Andy Woodruff 对 FA 地图的分析](https://andywoodruff.com/posts/2024/flightaware-maps/)。

---

## 1. 设计目标

| 层级 | 目标 |
|------|------|
| 底图 | 低对比、不抢戏；默认深色样式 |
| 叠加层 | 天气/网格等背景信息，低于动态目标 |
| 飞机 | 单一暖色、统一尺寸、可形成「流动纹理」 |
| 枢纽 | 与飞机两套视觉语言；远 zoom 分级显示 |
| 标注 | 底图地名始终可读（飞机/枢纽插在地名层之下） |

---

## 2. 图层顺序（自下而上）

```
底图瓦片
  → 地形晕渲（可选 hillshade）
  → 暗色底图对比度遮罩（仅深色 style，见 §3）
  → 天气网格 fill / line
  → AQI / 风温等可选层
  → 航班高度晕染圈（flight-altitude-glow）
  → 飞机图标（flight-points）
  → 枢纽圆点 + IATA 标签（airport-points / airport-labels）
  → 选中/高亮枢纽（airport-highlight-*）
  → 底图自带地名 symbol 层
```

实现要点：飞机与枢纽均通过 `beforeId` 插入到样式中第一个 `symbol` 层之前；**后添加**的层更靠近地名层，因此枢纽在飞机之上。

相关常量见 `client/src/config/mapVisualTheme.ts`。

---

## 3. 底图

- **默认**：`streets-v2-dark`（可由 `VITE_MAP_DEFAULT_STYLE` 覆盖）。
- **对比度**：对暗色 style 叠加全球 `fill` 遮罩（`basemap-dim`），`fill-opacity` 随 zoom 约 5%–14%，略微压暗陆地与海洋，不修改第三方 style JSON。
- **标注**：MapTiler / Stadia 可配置双语；OpenFreeMap 保留原生 `text-field`，仅统一 Noto 字体。

---

## 4. 飞机

| 项 | 约定 |
|----|------|
| 颜色 | 全图统一琥珀金 `#FFBF00`（`AIRCRAFT_AMBER`） |
| 描边 | SVG `stroke #140f08` + MapLibre `icon-halo`（约 1.25px） |
| 尺寸 | zoom 3–14 约在 `0.09–0.11`，差异小，避免远近「点阵感」不一致 |
| 旋转 | `icon-rotate` ← `heading`，`icon-rotation-alignment: map` |
| 重叠 | `icon-allow-overlap` + `icon-ignore-placement`，密集区保留连续纹理 |
| 地面/空中 | 同色；高度 &gt; 100ft 用空中图标，否则地面图标 |

配置：`client/src/config/aircraftColors.ts`（自动模式，无用户选色面板）。

### 4.1 选中 / 标记 / 悬停（feature-state）

- 实现：`client/src/utils/flightFeatureState.ts`，由 `mapUpdateScheduler` 合并刷新。
- 图层 paint 使用 `["feature-state", "selected"]` / `marked` / `hover`，**避免**为高亮频繁 `setData` 整源。
- 标记圈 filter：`client/src/utils/mapFilters.ts`；无标记时 `HIDE_ALL_FEATURES_FILTER = false`，**禁止** `flight_id === ""` 以免误匹配。

## 5. 枢纽机场

### 5.1 与 FlightAware 的关系（颜色是否按航班量？）

**FlightAware 官方说明**（Nearby Airports）：机场是否显示主要由 **机场规模（如每周航班量）** 与 **当前 zoom** 共同决定——缩放到全美只见 LAX 级枢纽，缩到加州才出现 SNA/SAN，再放大才有更小机场。**公开文档未写明「圆点颜色 = 实时航班数」**。

FA 地图上枢纽多为 **高饱和实心圆（绿/蓝等）+ 粗体 IATA（黄/黑 halo）**，用于与飞机图层区分，而非与飞机共用琥珀色。

**Sky-Trace 实现**：

- **显示数量**：按当前快照中起降计数排序，再按 zoom 截断上限（见下表），理念对齐 FA「远少近多」，数值为本项目估算而非 FA 官方阈值。
- **圆点颜色（绿/蓝/紫三档）**：在**当前可见枢纽集合内**按航班量分位数划分 tier 1/2/3，便于一眼区分繁忙程度；这是本项目的增强，**不是** FlightAware 已证实的官方规则。

### 5.2 远 zoom 显示上限（约 5000km 视角）

初始 zoom 多为 3–4。当前上限（`client/src/utils/mapHubDisplay.ts`）：

| zoom | 最多显示枢纽数 |
|------|----------------|
| &lt; 3 | 16 |
| 3 ≤ zoom &lt; 4 | 28 |
| 4 ≤ zoom &lt; 5 | 48 |
| 5 ≤ zoom &lt; 6 | 68 |
| 6 ≤ zoom &lt; 7 | 88 |
| ≥ 8 | 全部（约 98） |

无航班量的枢纽按 IATA 字母序作为 tie-break，避免顺序抖动。

### 5.3 样式

| 元素 | 值 |
|------|-----|
| Tier 1 圆 | `#22E584` |
| Tier 2 圆 | `#38BDF8` |
| Tier 3 圆 | `#C084FC` |
| 圆描边 | `#0f172a`，2.5px |
| IATA 文字 | `#FDA4AF`（玫瑰粉）+ `#312E81` indigo halo 2.5px，与琥珀金飞机区分 |

## 6. 右侧面板

| Tab | 组件 | 说明 |
|-----|------|------|
| 航班 | `flight/FlightListPanel` + `FlightFilterPanel` | 搜索、高级筛选、虚拟列表 |
| **枢纽** | `HubAirportListPanel` | DB 枢纽；中文名、离港/到港、定位、时刻表 |
| 标记 | `MarkedFlightsPanel` | 本地标记列表 |
| 详情 | `FlightDetailCard` | 选中航班 |
| 时刻 | `SchedulePanel` | 枢纽时刻表 |

枢纽中文名：`client/src/data/airportDisplay.ts`（城市映射 + 同城多机场专名，如 PEK/PKX）。

枢纽航班聚合：`client/src/utils/hubAirportList.ts`（按 `departure_airport` / `arrival_airport` 匹配当前快照）。

---

## 7. 天气网格（GRD）

- 后端 `GET /api/v1/weather-grid` 或前端由航班 + `datahub/weather` 回退生成。
- 以 **多边形 fill + outline** 展示 5°×5° 单元，非散点。
- 图层在飞机之下（`ensureWeatherGridLayers` 使用与飞机相同的 `beforeId` 锚点，且先于飞机添加）。

## 10. 航班筛选（右侧面板）

| 类别 | 能力 |
|------|------|
| 状态 | 全部 / 空中 / 地面 |
| 高度 | 最低–最高 ft，预设：地面 / 巡航 / 高空 |
| 航速 | 最低–最高 kts |
| 飞行类型 | 民航客机、通航、直升机、无人机、轻型（OpenSky `aircraft_category`） |
| 机型 | 多选 OpenSky 类别（Large / Heavy / Rotorcraft 等） |
| 地区 | 四级：国家 → 省州 → 城市 → 区县（中/美/日/德有嵌套数据，见 `geoNested.ts`） |
| 范围 | 领空 / 出发 / 到达 |

实现：`client/src/stores/flight.ts`、`client/src/utils/flightFilters.ts`、`client/src/components/flight/FlightFilterPanel.vue`。

## 12. 与 FlightAware / FlightRadar24 刷新周期对照（调研摘要）

| 平台 | 文档/官方说明 | 要点 |
|------|----------------|------|
| **FlightAware** | [FAQ](https://www.flightaware.com/about/faq.rvt) | 页面约 **每分钟 1–4 次** 更新；注册用户更频繁；展示数据通常 **延迟约 30 秒**；地图动画位置可能落后真值 **1–2 分钟**。 |
| **FlightRadar24** | [Blog 2025-03](https://www.flightradar24.com/blog/inside-flightradar24/supercharging-flightradar24s-data-display/) | 实时流约 **2–3 秒** 刷新（此前约 8 秒）；ADS-B 上报约 **4–5 秒**；地图在两次位置间插值平滑移动。 |
| **Sky-Trace** | `server` 配置 | 开发环境 OpenSky 轮询约 **30–90s**（可配置）；Release **10s**；WebSocket 在每次采集完成后推送 `snapshot`（`WS_HEARTBEAT_SECONDS` 默认 20s 保活）。 |
