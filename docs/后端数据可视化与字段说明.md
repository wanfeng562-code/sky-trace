# 后端数据可视化与字段说明

## 1. 目的
本文件用于回答以下问题：
- 通过接口获取到的数据都是什么。
- 这些数据有什么用。
- 在前后端中应该如何使用这些数据。
- 如何直接看到后端返回的具体数据。

## 2. 可直接访问的后端入口

说明：以下地址默认使用 `8000` 端口。若 `8000` 已被占用，请将地址中的端口统一替换为你实际启动端口（例如 `8001`）。

### 2.1 可视化页面（推荐先看）
- 地址（8000）：`http://127.0.0.1:8000/debug/flights-dashboard`
- 地址（8001 示例）：`http://127.0.0.1:8001/debug/flights-dashboard`
- 作用：
  - 自动轮询后端接口并展示航班摘要指标。
  - 展示当前航班列表。
  - 以散点图方式展示经纬度分布。
  - 展示原始 JSON，便于核对“接口真实返回”。

### 2.2 数据接口
- 航班列表：`GET /api/v1/flights?page=1&page_size=500`
- 航班详情：`GET /api/v1/flights/{flight_id}`
- 航班轨迹：`GET /api/v1/flights/{flight_id}/track`
- 航班汇总：`GET /api/v1/flights/summary`

## 3. 数据含义与用途

### 3.1 航班列表与详情字段（核心业务字段）
| 字段          | 含义          | 常见用途                     |
| ------------- | ------------- | ---------------------------- |
| `flight_id`   | 航班唯一标识  | 列表跳转详情、轨迹查询主键   |
| `callsign`    | 航班呼号      | 航班搜索、展示标签           |
| `lat` / `lon` | 当前纬度/经度 | 地图打点、位置更新、区域过滤 |
| `heading`     | 航向角（度）  | 飞机图标朝向、运动趋势判断   |
| `speed_kts`   | 速度（节）    | 速度面板、异常速度检测       |
| `altitude_ft` | 高度（英尺）  | 高度分层、爬升/下降分析      |
| `updated_at`  | 数据更新时间  | 新鲜度判断、超时告警         |

### 3.2 轨迹点字段（回放与趋势）
| 字段          | 含义         | 常见用途             |
| ------------- | ------------ | -------------------- |
| `ts`          | 轨迹采样时间 | 回放时间轴、轨迹排序 |
| `lat` / `lon` | 轨迹点位置   | 绘制轨迹线           |
| `altitude_ft` | 轨迹点高度   | 爬升/下降趋势图      |
| `speed_kts`   | 轨迹点速度   | 加减速分析           |

### 3.3 汇总字段（看全局状态）
| 字段                                    | 含义          | 常见用途         |
| --------------------------------------- | ------------- | ---------------- |
| `total`                                 | 当前航班总数  | 总体负载监控     |
| `avg_altitude_ft`                       | 平均高度      | 空域层级总体判断 |
| `min_altitude_ft` / `max_altitude_ft`   | 最低/最高高度 | 极值监控         |
| `avg_speed_kts`                         | 平均速度      | 运行态势评估     |
| `min_speed_kts` / `max_speed_kts`       | 最低/最高速度 | 异常检测         |
| `lat_min`/`lat_max`/`lon_min`/`lon_max` | 覆盖范围边界  | 地图视窗自适应   |
| `latest_updated_at`                     | 最近刷新时间  | 数据时效性监控   |

## 4. 如何使用这些数据

### 4.1 后端调试阶段
1. 先看 `/debug/flights-dashboard`，确认接口和字段是否稳定。
2. 再看页面里的 Raw JSON，确认每个字段类型与单位。
3. 用 `/api/v1/flights/summary` 做快速健康检查。

### 4.2 前端对接阶段
1. 列表页绑定 `/api/v1/flights`。
2. 点击航班后，用 `flight_id` 请求详情和轨迹。
3. 地图视图以 `lat/lon/heading/speed_kts/altitude_ft` 渲染动态点。
4. 仪表盘顶部指标使用 `/api/v1/flights/summary`。

### 4.3 报告与演示阶段
1. 使用 dashboard 页面截图展示“可视化结果”。
2. 使用 Raw JSON 截图展示“真实返回结构”。
3. 使用 summary 数值展示“全局态势指标”。

## 6. 启动方式
```powershell
cd server
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

启动后访问：
- `http://127.0.0.1:8000/debug/flights-dashboard`

若 `8000` 端口被占用，可改用：

```powershell
cd server
python -m uvicorn app.main:app --host 127.0.0.1 --port 8001
```

改用 `8001` 后，访问地址同步改为：
- `http://127.0.0.1:8001/debug/flights-dashboard`

## 7. 常见现象说明

- 访问 `http://127.0.0.1:8000/`：已自动重定向到 dashboard 页面（`/debug/flights-dashboard`）。
- 日志出现 `/hybridaction/... 404`：通常是本机其他程序或浏览器插件发起的探测请求，不是本项目接口路径，不影响系统主流程。
