## 1. 部署架构总览

```
用户浏览器
    │
    ▼  HTTPS (443)
┌──────────────────────────────────────────────┐
│  Cloudflare Pages（全球 CDN）                  │
│  https://sky-trace.pages.dev/                 │
│  Vue3 + TypeScript + Vite 静态产物             │
│  由 Gitee 仓库 main 分支自动构建               │
└───────────────────┬──────────────────────────┘
                    │  HTTPS REST / WSS WebSocket
                    │  端口 8443
                    ▼
┌──────────────────────────────────────────────┐
│  AutoDL 云服务器（西部云节点）                  │
│  FastAPI + Uvicorn，监听 0.0.0.0:6006         │
│  AutoDL 端口映射：内部 6006 → 外部 8443 (SSL) │
│  SQLite WAL + 三层统一采集管道                 │
└───────────────────┬──────────────────────────┘
                    │  HTTP（国内可访问）
                    ▼
  ┌─────────────────────────────────────┐
  │  外部数据 API                        │
  │  OpenSky Network / OpenWeatherMap   │
  │  AirLabs / FR24（经 CF Worker 代理） │
  └─────────────────────────────────────┘
```

**关键约束**：
- AutoDL 服务器位于中国大陆（西部云节点），可访问 OpenSky、OWM、AirLabs
- FR24 走 Cloudflare Worker 代理（`*.workers.dev`）——AutoDL 因 GFW 屏蔽 `*.workers.dev` 导致 FR24 不可用，其余数据源均正常
- DataV 地图数据（`geo.datav.aliyun.com`）对 CF Pages（海外域名）返回 403；前端已内置 `cn-admin1.json` 兜底

### 2.2 代码上传方式

**方式 A：通过 Gitee 克隆（推荐）**

```bash
# 在 AutoDL 终端（JupyterLab 或 SSH）执行
cd /root
git clone https://gitee.com/<你的用户名>/sky-trace.git Sky-Trace
cd Sky-Trace
```

**方式 B：SCP 上传压缩包**

```powershell
# 在本地执行（需知道 AutoDL SSH 端口）
scp -P <ssh_port> server.zip root@<实例IP>:/root/
```

### 2.3 虚拟环境与依赖安装

```bash
cd /root/Sky-Trace/server

python -m venv .venv
source .venv/bin/activate

pip install -r requirements.txt
```

> AutoDL 预装了 Python 3.10+，通常无需手动安装 Python 本体。

### 2.4 服务器 .env 配置

在 `server/` 目录新建 `.env`（不要提交到 git，已在 `.gitignore` 中）：

```ini
APP_NAME=Sky-Trace-Server
APP_ENV=production
APP_PROFILE=development       # 保持 development，控制采集频率
APP_HOST=0.0.0.0              # 必须监听全网卡，否则 AutoDL 代理无法转发
APP_PORT=6006                 # AutoDL 映射到外部 8443 的内部端口
LOG_LEVEL=INFO

# OpenSky OAuth2（推荐，4000 积分/天）
OPENSKY_CLIENT_ID=你的ClientID
OPENSKY_CLIENT_SECRET=你的ClientSecret
OPENSKY_BBOX=                 # 留空 = 全球模式（~6000 架次）

# 商业层 AirLabs（航班号/机型/时刻表）
AIRLABS_API_KEY=你的AirLabsKey

# 气象数据 OpenWeatherMap
OPENWEATHER_API_KEY=你的OWMKey

# FR24 代理（可选，AutoDL 因 GFW 无法访问 *.workers.dev，留空则跳过）
FR24_PROXY_URL=

# CORS：允许 CF Pages 前端跨域访问
CORS_ALLOW_ORIGINS=https://sky-trace.pages.dev,http://localhost:5173

# SQLite 数据库路径
SQLITE_PATH=./data/sky_trace.db

# WebSocket 心跳
WS_HEARTBEAT_SECONDS=20
```

> ⚠️ `APP_HOST=0.0.0.0` 是必填项。若设置为 `127.0.0.1`，AutoDL 的端口映射代理将无法到达 Uvicorn。

### 2.5 启动服务

```bash
cd /root/Sky-Trace/server
source .venv/bin/activate

# 方式 A：使用 run.py（自动读取 .env 中的 APP_HOST / APP_PORT）
python run.py

# 方式 B：直接指定
python -m uvicorn app.main:app --host 0.0.0.0 --port 6006
```

**后台常驻（nohup）**：

```bash
nohup python run.py > /root/sky-trace.log 2>&1 &
echo $! > /root/sky-trace.pid
echo "后端已在后台启动，PID: $(cat /root/sky-trace.pid)"
```

**查看日志**：

```bash
tail -f /root/sky-trace.log
```

**停止服务**：

```bash
kill $(cat /root/sky-trace.pid)
```

## 3. 前端部署（Cloudflare Pages）

### 3.1 仓库连接

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 进入 **Pages → Create a project → Connect to Git**
3. 授权连接 Gitee 仓库（需通过 GitHub 中转或使用 Direct Upload；若使用 GitHub 镜像则直接选 GitHub）
4. 选择仓库 `Sky-Trace`，分支 `main`

### 3.2 构建配置

| 设置项           | 值                  |
| ---------------- | ------------------- |
| 框架预设         | Vue                 |
| 根目录           | `client`            |
| 构建命令         | `npm run build`     |
| 构建输出目录     | `dist`              |
| Node.js 版本     | 20.x（在环境变量中设置 `NODE_VERSION=20`） |

### 3.3 环境变量（`client/.env.production`）

本项目已将生产环境变量提交到仓库 `client/.env.production`，Vite 构建时自动读取：

```ini
VITE_APP_NAME=Sky-Trace
VITE_API_BASE_URL=https://u786849-b2d5-be7b9d68.westc.seetacloud.com:8443/api/v1
VITE_WS_URL=wss://u786849-b2d5-be7b9d68.westc.seetacloud.com:8443/api/v1/ws/flights
VITE_ENABLE_MOCK=false
VITE_MAP_DEFAULT_PROVIDER=maptiler
VITE_MAP_DEFAULT_STYLE=streets-v2-dark
```

> `ws://` 须升级为 `wss://`（WebSocket Secure），因为 CF Pages 为 HTTPS，混合内容（HTTP WebSocket）会被浏览器阻止。

如需在 CF Pages 控制台覆盖环境变量（而不走 `.env.production`），在 **Settings → Environment variables** 中设置同名变量即可。

## 4. CORS 跨域配置

CF Pages（`sky-trace.pages.dev`）向 AutoDL 后端发起跨域请求，后端必须在响应头中声明允许来源。

FastAPI CORS 中间件已在 `server/app/main.py` 中配置，对应 `.env` 字段：

```ini
CORS_ALLOW_ORIGINS=https://sky-trace.pages.dev,http://localhost:5173
```

> 开发时加入 `http://localhost:5173` 允许本地前端直连云后端（混合联调）。  
> 多个来源用英文逗号分隔，无空格。

## 6. 已知限制与注意事项

### 6.1 FR24 数据缺失（GFW + CF Worker 冲突）

**现象**：`GET /api/v1/datahub/status` 中 `fr24.disabled=true` 或 `fr24.cache_count=0`

**根因**：AutoDL 所在的中国大陆网络无法访问 Cloudflare Worker 域名（`*.workers.dev` 被 GFW 屏蔽）。

**影响**：FR24 补充商业层数据不可用（约占商业航班信息的 10%–20%）；OpenSky / AirLabs 数据均正常，实时位置、机型、航班号不受影响。

**可选解决方案**（暂未实施）：
- 将 CF Worker 绑定自定义域名（需付费 CF 账号）；
- 在 AutoDL 上配置 HTTP_PROXY 通过代理访问 Worker。

### 6.2 DataV 地图 API 海外域名 403

**现象**：从 `sky-trace.pages.dev` 调用 `geo.datav.aliyun.com` 返回 403，省级边界降级为矩形。

**根因**：DataV 阿里云 API 屏蔽了海外域名的跨域请求。

**已修复**（commit `b5ca87c`）：DataV 返回 403 后，前端自动回退到已打包的 `cn-admin1.json`（Natural Earth 数据，32 个大陆省份），国级轮廓用所有省份合并生成。HK/TW/MO 因不在 `cn-admin1.json` 中，回退为 bbox 矩形（这三个区域面积极小，误差视觉可接受）。

### 6.3 AutoDL 实例重启后地址不变

AutoDL 实例停机重启后，公网 URL **不变**（URL 与实例 ID 绑定）。但 .env 文件保存在实例存储空间中，不会丢失。重启后执行：

```bash
cd /root/Sky-Trace/server
source .venv/bin/activate
nohup python run.py > /root/sky-trace.log 2>&1 &
```

### 6.4 SQLite 并发限制

本项目使用单文件 SQLite（WAL 模式），满足当前单实例场景。若未来需要多副本横向扩展，需迁移至 PostgreSQL。
