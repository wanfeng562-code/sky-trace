## 7. 结论与建议

### 互补策略

```
OpenSky   →  主力实时位置（icao24/lat/lon/alt/speed/heading）
AirLabs   →  富集层（dep/arr/aircraft_type/airline/reg_number），单次调用覆盖所有航班
FR24 SDK  →  备援补充（OpenSky离线/429时切换），但应控制频率防止封IP
```

### AirLabs 额度优化建议

- 一次 `/flights` 调用（不带bbox）可获取全球所有可见航班，同时获得位置+富集数据
- 额度：1000次/月 ≈ 33次/天，配合24h刷新间隔仅需1次/天
- 若改为实时位置补充（90s一次）则需 960次/天 × 30天 = 28800次/月，超出免费额度
- **推荐**：保持24h一次的商业层富集，不用于实时位置

### FlightRadar24 使用注意

> **⚠ 当前状态（2026-05-09 测试）：FlightRadar24 返回 403 Forbidden**
> `data-cloud.flightradar24.com/zones/fcgi/feed.js` 接口已对非官方 SDK 封锁，
> FlightRadarAPI 库在当前版本下无法正常工作。

- 非官方SDK（JeanExtreme002/FlightRadarAPI），无SLA保证
- FR24 已于 2025 年前后开始限制非官方 SDK 访问（`feed.js` 返回 403）
- 官方提供商业API：https://fr24api.flightradar24.com/ （需申请，付费）
- **当前建议**：不纳入数据源策略，等待 SDK 作者更新或转用 FR24 官方 API
- 若 SDK 将来恢复可用，建议轮询间隔 ≥ 60s，避免触发 IP 封禁
