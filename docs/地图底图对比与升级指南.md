## 1. 技术背景

Sky-Trace 前端使用 **MapLibre GL JS** 渲染地图，该库是 Mapbox GL JS v1 的开源分叉，完全兼容 Mapbox Style Specification。切换底图只需修改 `style` URL——**无需更换地图渲染库本身**。

```
┌──────────────────────────────────────────┐
│       MapLibre GL JS（渲染引擎）           │
│  接受任意符合 Mapbox Style Spec 的 URL    │
└───────────────┬──────────────────────────┘
                │ style JSON
    ┌───────────┼──────────────────────┐
    ▼           ▼                      ▼
OpenFreeMap  MapTiler Cloud     Stadia Maps / Mapbox ...
（当前）      （PlaybackView      （第三方托管服务）
              已集成）
```

## 3. 详细评测

### 3.1 OpenFreeMap（当前）

**官网**：https://openfreemap.org

**底图样式预览**：

| 样式名     | 风格描述       | 适用场景       |
| ---------- | -------------- | -------------- |
| `liberty`  | 浅色，OSM 数据 | ✅ 当前使用     |
| `bright`   | 高对比度，浅色 | 数据可读性优先 |
| `positron` | 极简灰白       | 数据叠加层优先 |

**Style URL 格式**：
```
https://tiles.openfreemap.org/styles/{style_name}
```

**优点**：
- 完全免费，无 API Key，无用量限制
- 无隐私顾虑，不向第三方上报用户坐标
- 适合学习项目和展示

**缺点**：
- 样式设计相对基础，视觉精细度不及商业方案
- 无专业航空图层（等高线、气压带等）
- 瓦片服务器稳定性无 SLA 保障
- 暗色模式样式质量一般

**适用结论**：课程作业展示、Demo、对视觉要求低的场景 ✅

---

### 3.2 MapTiler Cloud

**官网**：https://cloud.maptiler.com  
**注册**：https://cloud.maptiler.com/auth/widget?mode=login  
**免费额度**：10 万 tile 请求 / 月（约 3000 次地图加载）

**Style URL 格式**：
```
https://api.maptiler.com/maps/{style_id}/style.json?key={YOUR_KEY}
```

**推荐样式清单**：

| style_id           | 风格描述                 | FR24/FlightAware 对标     |
| ------------------ | ------------------------ | ------------------------- |
| `streets-v2`       | 现代浅色街道图，标注清晰 | 接近 FlightAware          |
| `dataviz`          | 数据可视化专用，中性浅色 | **最适合 Sky-Trace**      |
| `bright`           | 高饱和街道图             | 视觉活跃                  |
| `satellite`        | 卫星影像                 | 高真实感                  |
| `satellite-hybrid` | 卫星 + 矢量标注          | 最像 FlightAware 卫星模式 |
| `topo-v2`          | 地形等高线               | 山区航班展示              |
| `openstreetmap`    | 标准 OSM 渲染            | 数据最全                  |
| `backdrop`         | 极简城市轮廓             | AQI 叠加优先              |

**暗色系样式**：

| style_id          | 特点                                 |
| ----------------- | ------------------------------------ |
| `dataviz-dark`    | **推荐**，深色背景，航班点高亮对比强 |
| `streets-v2-dark` | 深色街道，类 FR24 夜间模式           |

**优点**：
- 样式精细，专业设计，视觉效果接近 FR24
- 支持浅色/深色/卫星切换
- 项目已有 `PlaybackView.vue` 集成，现有 `VITE_MAPTILER_KEY` 变量复用
- 免费额度对课程项目绰绰有余
- 支持中文地名（包含 `name:zh`）

**缺点**：
- 需要注册账号，有 Key 泄露风险（须加入 `.gitignore`）
- 月超 10 万 tile 开始计费

**适用结论**：**首推方案**，兼顾视觉质量与成本 ✅✅✅

### 3.3 Stadia Maps

**官网**：https://stadiamaps.com  
**注册**：https://client.stadiamaps.com/signup/  
**免费额度**：20 万 map tile 请求 / 月

**Style URL 格式**：
```
https://tiles.stadiamaps.com/styles/{style_id}/style.json?api_key={YOUR_KEY}
```

**推荐样式清单**：

| style_id              | 风格描述                   | 特点                 |
| --------------------- | -------------------------- | -------------------- |
| `alidade_smooth`      | 柔和浅色                   | 低噪声，利于数据叠加 |
| `alidade_smooth_dark` | **暗色系最佳**，深蓝灰底色 | 最像 FR24 深色模式   |
| `alidade_satellite`   | 卫星影像 + 标注            | 高真实感             |
| `stamen_toner`        | 极简黑白                   | 高对比度             |
| `stamen_terrain`      | 地形渲染                   | 山地可视化           |
| `osm_bright`          | 标准 OSM 浅色              | 数据最完整           |

**优点**：
- `alidade_smooth_dark` 是所有免费/低价方案中最接近 FR24 深色效果的样式
- 免费额度高于 MapTiler
- 支持 Retina 高分辨率

**缺点**：
- 中文地名支持有限（`name:zh` 覆盖率不及 MapTiler）
- 样式数量少于 MapTiler
- 国内访问速度一般

**适用结论**：追求深色 FR24 风格的最优选 ✅✅

### 3.4 Mapbox GL JS

**官网**：https://www.mapbox.com  
**注册**：https://account.mapbox.com/auth/signup/  
**免费额度**：5 万 地图加载 / 月（比 tile 更严格）

> ⚠️ **注意**：Mapbox GL JS v2+ 引入了专有 SDK 协议，不能直接用 MapLibre GL JS 加载其 style URL。需要通过兼容的 TileJSON 端点或继续使用 MapLibre GL JS（可访问 v1 公开端点）。

**推荐方式**：使用 Mapbox 瓦片 API，配合 MapLibre GL JS 自定义 style JSON：
```json
{
  "version": 8,
  "sources": {
    "mapbox-streets": {
      "type": "vector",
      "url": "mapbox://mapbox.mapbox-streets-v8",
      "access_token": "YOUR_MAPBOX_TOKEN"
    }
  },
  ...
}
```

**优点**：
- 商业级最高质量，FR24 和 FlightAware 采用的技术
- 卫星影像分辨率最高
- 丰富的自定义样式工具（Mapbox Studio）

**缺点**：
- 配置复杂，不能直接用 style URL
- 免费额度最少（5 万 load/月）
- 国内访问速度慢

**适用结论**：商业产品推荐，课程项目配置复杂度偏高 ⚠️

### 3.5 Esri Basemap Styles（ArcGIS）

**官网**：https://developers.arcgis.com/documentation/mapping-apis-and-services/maps/services/basemap-layer-service/  
**注册**：https://developers.arcgis.com/sign-up/  
**免费额度**：200 万 tile / 月（最慷慨）

**Style URL 格式**：
```
https://basemapstyles-api.arcgis.com/arcgis/rest/services/styles/v2/styles/{style_id}?token={YOUR_KEY}
```

**推荐样式**：

| style_id                  | 特点                        |
| ------------------------- | --------------------------- |
| `arcgis/streets`          | 标准街道                    |
| `arcgis/dark-gray`        | 深灰底色                    |
| `arcgis/navigation-night` | **黑色夜间导航，最像 FR24** |
| `arcgis/imagery/hybrid`   | 卫星 + 道路标注             |
| `arcgis/ocean`            | 海洋专题图                  |

**优点**：
- 免费额度最大（200 万/月）
- Esri 专业地理数据质量高
- FlightAware 采用 Esri 作为底图

**缺点**：
- 对 MapLibre GL JS 的兼容性需要额外处理（Esri 有专门的 `@arcgis/map-components` SDK）
- 中文地名支持一般
- 国内访问速度较慢

**适用结论**：想体验 FlightAware 风格首选，但配置需要额外工作量 ⚠️

### 3.6 OpenStreetMap + 自部署 Tileserver-GL

**适用场景**：需要完全离线、完全自控、无任何费用

**部署方式**：
```bash
# 下载 mbtiles（以中国区域为例）
wget https://download.geofabrik.de/asia/china-latest.osm.pbf
# 使用 tileserver-gl 提供矢量瓦片
docker run -p 8080:80 -v $(pwd):/data maptiler/tileserver-gl
```

**优点**：完全离线、零成本  
**缺点**：需要服务器、设置复杂，样式需自定义

**适用结论**：仅推荐有服务器部署经验的场景

## 6. 升级操作手册

### 6.1 切换到 MapTiler Cloud（推荐）

**步骤 1**：注册并获取 API Key

1. 前往 https://cloud.maptiler.com/auth/widget?mode=login 注册免费账号
2. 登录后进入 **Account → API Keys**
3. 复制 Default Key（格式：32 位字母数字字符串）

**步骤 2**：配置环境变量

在 `client/.env.development` 末尾追加：
```dotenv
VITE_MAPTILER_KEY=你的MapTilerKey粘贴在这里
```

确认 `.gitignore` 中已包含 `.env.development`（防止 Key 泄露）：
```gitignore
client/.env.development
client/.env.production
client/.env.local
```

**步骤 3**：修改 MapView.vue

找到 `client/src/views/MapView.vue` 中的 `initMap()` 函数，修改 `style` 字段：

```typescript
// 在文件顶部添加（script setup 区域开头）
const MAPTILER_KEY = import.meta.env.VITE_MAPTILER_KEY as string;

// 在 initMap() 中修改 style：
const mapInstance = new maplibregl.Map({
  container: mapContainer.value,
  // 替换此行：
  style: `https://api.maptiler.com/maps/dataviz-dark/style.json?key=${MAPTILER_KEY}`,
  // 其余参数不变
  center: [113.2644, 23.1291],
  zoom: 4,
  attributionControl: true,
});
```

**可选样式 URL 备选**（仅修改 style 行即可切换）：

```typescript
// 浅色专业风（类 FlightAware）
style: `https://api.maptiler.com/maps/dataviz/style.json?key=${MAPTILER_KEY}`

// 深色 FR24 风（推荐）
style: `https://api.maptiler.com/maps/dataviz-dark/style.json?key=${MAPTILER_KEY}`

// 卫星 + 标注（类 FlightAware 卫星模式）
style: `https://api.maptiler.com/maps/satellite-hybrid/style.json?key=${MAPTILER_KEY}`

// 街道图
style: `https://api.maptiler.com/maps/streets-v2/style.json?key=${MAPTILER_KEY}`

// 深色街道
style: `https://api.maptiler.com/maps/streets-v2-dark/style.json?key=${MAPTILER_KEY}`
```

**步骤 4**：验证

```bash
cd client
npm run dev
```

打开浏览器 → 进入主地图页 → 地图应加载为新样式。若样式不变，检查浏览器 Network 面板确认 style URL 是否 200 OK。

## 8. 故障转移机制

```
启动时（determineInitialStyle）：
  MAPTILER_KEY 不为空
    → 探测 /maptiler-proxy/maps/dataviz-dark/style.json
      成功 → 使用 MapTiler，provider = "maptiler"
      失败 → 尝试 Stadia
  STADIA_KEY 不为空
    → 使用 Stadia alidade_smooth_dark，provider = "stadia"
  否则 → 使用 OpenFreeMap liberty，provider = "openfreemap"

运行时（map.on("error")）：
  HTTP 403 / 429 + provider == "maptiler" + STADIA_KEY 不为空
    → 自动切换到 Stadia，显示橙色 toast（8 秒可点击关闭）
    → map.setStyle(stadiaUrl) + reinitLayers
```

**前端图层切换面板**（左上角 ⊞ 图层 按钮）：
- 底图样式：MapTiler 4 款（Dataviz Dark/Light、Hybrid、Topo）或 Stadia 3 款
- 图层叠加：AQI 热力图、风速风向、温度分布、航班密度热力图
- AQI 圆点（原有功能，独立开关）

---

**Q1：切换底图后双语地名（英文 + 中文）功能还有效吗？**

`applyBilingualLabels()` 函数遍历所有 symbol 图层并替换 `text-field`，适用于任何包含 `name:zh-Hans`/`name:zh` 属性的矢量底图（MapTiler、Stadia 均支持）。切换底图后双语显示自动生效，无需修改该函数。

**Q2：切换底图后地形晒渲（hillshade）还能用吗？**

是的。项目中的 hillshade 使用 AWS 免费 DEM 源（`s3.amazonaws.com/elevation-tiles-prod`），与底图提供商无关：

```typescript
mapInstance.addSource("terrain-dem", {
  type: "raster-dem",
  tiles: ["https://s3.amazonaws.com/elevation-tiles-prod/terrarium/{z}/{x}/{y}.png"],
  ...
});
```

切换底图不影响地形渲染层。

---

**Q3：MapTiler 的 dataviz-dark 样式本身就有深色地图，和 AQI 叠加层会不会冲突？**

不会冲突。AQI 叠加层使用 MapLibre 的独立 GeoJSON source，颜色通过 `circle-color` step 表达式控制，与底图样式完全独立。深色底图反而让 AQI 色圈对比度更高、更易分辨。

---

**Q4：PlaybackView 已经在用 `VITE_MAPTILER_KEY`，是同一个 Key 吗？**

是的。PlaybackView.vue 中已引用：

```typescript
const MAPTILER_KEY = import.meta.env.VITE_MAPTILER_KEY as string;
// style: `https://api.maptiler.com/maps/streets/style.json?key=${MAPTILER_KEY}`
```

只需在 `.env.development` 中填写一次 `VITE_MAPTILER_KEY`，两个视图同时生效。

---

**Q5：MapTiler 免费额度 10 万 tile/月 够用吗？**

课程展示场景下绰绰有余。一次地图加载约消耗 20-60 个 tile（取决于缩放级别和屏幕尺寸）。10 万 tile ≈ 约 1700-5000 次地图加载，足够日常开发、演示、评测使用。

---

**Q6：如何在不修改代码的情况下动态切换底图样式？**

可以将 style URL 模板化，通过环境变量控制：

```dotenv
# .env.development
VITE_MAP_STYLE_URL=https://api.maptiler.com/maps/dataviz-dark/style.json?key=你的Key
```

```typescript
// MapView.vue initMap() 中
const mapInstance = new maplibregl.Map({
  style: import.meta.env.VITE_MAP_STYLE_URL as string,
  ...
});
```

这样切换样式只需改 `.env` 文件，无需修改 Vue 代码。
