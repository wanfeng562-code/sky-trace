# Sky-Trace 地图页面配置说明文档

> 最后更新：**2026-05-18**。介绍 `MapView.vue` / `PlaybackView.vue` 底图与代理。  
> **生产环境**：瓦片经后端 `/api/v1/tiles/*` 代理（`mapBasemap.ts`）；**开发环境**：Vite 代理 `/maptiler-proxy` 等。图层与视觉见 [地图可视化设计说明.md](./地图可视化设计说明.md)。

---

## 1. 地图库

Sky-Trace 使用 **[MapLibre GL JS](https://maplibre.org/) v4.7.1** 作为地图渲染引擎，该库是 Mapbox GL JS 的开源分支，支持任意 tile 源和样式（Mapbox GL Style Spec v8）。

---

## 2. 三地图 Provider 架构

地图页面支持三个 provider，按优先级自动选择或回退：

```
VITE_MAP_DEFAULT_PROVIDER=maptiler
        │
        ▼
   MapTiler Cloud   ──（Key 无效 / 连接失败）──►  Stadia Maps  ──（无 Key）──►  OpenFreeMap
  （需 API Key）                                （需 API Key）                （无需 Key）
```

### 2.1 MapTiler Cloud（首选）

- 官网：https://cloud.maptiler.com
- 免费额度：10 万 map loads / 月
- 特点：样式丰富，质量最高，支持卫星影像和地形图
- 需要在 `.env` 中设置 `VITE_MAPTILER_KEY`

**可用样式：**

| 样式 ID | 中文名 | 说明 |
|---|---|---|
| `streets-v2-dark` | 街道深色 | 深色街道图，对比度高，**推荐作为飞行追踪默认底图** |
| `streets-v2` | 街道浅色 | 浅色街道图，地理信息丰富 |
| `outdoor-v2` | 户外地形 | 户外地形风格，高程可视化，适合低空飞行场景 |
| `satellite-hybrid` | 卫星影像 | 卫星 + 道路标注叠合 |
| `dataviz-dark` | 数据深色 | 极简深色，适合 BI 数据展示（低地理上下文） |
| `dataviz` | 数据浅色 | 极简浅色，适合 BI 数据展示 |

### 2.2 Stadia Maps（备用）

- 官网：https://stadiamaps.com
- 免费额度：非商业用途 + 署名即可免费
- 特点：样式精美，平滑过渡，适合展示应用
- 需要在 `.env` 中设置 `VITE_STADIA_KEY`

**可用样式：**

| 样式 ID | 中文名 | 说明 |
|---|---|---|
| `alidade_smooth_dark` | 深色平滑 | 深色低噪声，**Stadia 默认首选** |
| `alidade_smooth` | 浅色平滑 | 浅色平滑，低噪声 |
| `stamen_terrain` | 地形图 | 自然地形风格，高程晕渲 |
| `alidade_satellite` | 卫星影像 | 高分辨率卫星底图 |

### 2.3 OpenFreeMap（最终兜底）

- 官网：https://openfreemap.org
- 无需 API Key
- 特点：基于 OpenStreetMap 数据，完全免费开放

**可用样式：**

| 样式 ID | 中文名 | 说明 |
|---|---|---|
| `liberty` | 自然风格 | 自然色调，类传统地图，**默认** |
| `bright` | 明亮风格 | 高对比度，边界清晰 |
| `positron` | 简洁浅色 | 极简浅灰，适合密集数据叠加 |

## 3. 环境变量配置

所有地图相关配置均在 `client/.env` 中设置，修改后重启 Vite 开发服务器生效。

### API Keys

```dotenv
# MapTiler Cloud API Key
# 获取：https://cloud.maptiler.com/account/keys/
# 留空则跳过 MapTiler，回退到 Stadia
VITE_MAPTILER_KEY=<your-key>

# Stadia Maps API Key
# 获取：https://client.stadiamaps.com/
# 留空则最终使用 OpenFreeMap（无需 Key）
VITE_STADIA_KEY=<your-key>
```

### 地图默认配置

```dotenv
# 默认地图 provider
# 可选值：maptiler | stadia | openfreemap
VITE_MAP_DEFAULT_PROVIDER=maptiler

# 默认底图样式 ID（与上方 provider 匹配）
# MapTiler:    streets-v2-dark | streets-v2 | outdoor-v2 | satellite-hybrid | dataviz-dark | dataviz
# Stadia:      alidade_smooth_dark | alidade_smooth | stamen_terrain | alidade_satellite
# OpenFreeMap: liberty | bright | positron
VITE_MAP_DEFAULT_STYLE=streets-v2-dark

# 初始地图视口（中心点经度/纬度，缩放级别）
VITE_MAP_INIT_LNG=113.2644   # 广州经度
VITE_MAP_INIT_LAT=23.1291    # 广州纬度
VITE_MAP_INIT_ZOOM=4         # 全国视角
```

### 配置示例

**以 Stadia 深色风格为默认：**
```dotenv
VITE_MAP_DEFAULT_PROVIDER=stadia
VITE_MAP_DEFAULT_STYLE=alidade_smooth_dark
```

**以 OpenFreeMap 明亮风格为默认（无 Key 环境）：**
```dotenv
VITE_MAP_DEFAULT_PROVIDER=openfreemap
VITE_MAP_DEFAULT_STYLE=bright
```

**默认聚焦北京区域：**
```dotenv
VITE_MAP_INIT_LNG=116.3974
VITE_MAP_INIT_LAT=39.9093
VITE_MAP_INIT_ZOOM=6
```

## 4. 开发代理配置（vite.config.ts）

由于 MapLibre Web Worker 运行在 `blob:` URL 上下文中，无法直接请求外部 tile 域名（跨域），因此所有 tile 请求必须通过 Vite 开发代理转发。

`client/vite.config.ts` 中的代理配置：

```typescript
proxy: {
    "/maptiler-proxy": {
        target: "https://api.maptiler.com",
        rewrite: (path) => path.replace(/^\/maptiler-proxy/, ""),
        changeOrigin: true,
        secure: true,
    },
    "/stadia-proxy": {
        target: "https://tiles.stadiamaps.com",
        rewrite: (path) => path.replace(/^\/stadia-proxy/, ""),
        changeOrigin: true,
        secure: true,
    },
}
```

> **OpenFreeMap** 的 tile URL（`https://tiles.openfreemap.org/...`）直接在 `transformRequest` 中透传，无需代理，因为 OpenFreeMap 的样式文件和 tile 均支持 CORS。

---

## 5. `transformRequest` 实现原理

`transformRequest` 是 MapLibre 提供的 URL 转换钩子，拦截地图发出的所有请求（tile、字体、精灵图等），在发送前将外部 URL 重写为代理路径。

```typescript
function transformRequest(url: string): { url: string } {
    const origin = window.location.origin;   // 例如 http://localhost:5173
    if (url.startsWith("https://api.maptiler.com")) {
        return { url: url.replace("https://api.maptiler.com", `${origin}/maptiler-proxy`) };
    }
    if (url.startsWith("https://tiles.stadiamaps.com")) {
        return { url: url.replace("https://tiles.stadiamaps.com", `${origin}/stadia-proxy`) };
    }
    return { url };   // OpenFreeMap / 其他 URL 直接透传
}
```

**关键点：必须返回绝对 URL**（含协议 + 域名），不能使用相对路径（如 `/maptiler-proxy/...`）。原因是 MapLibre 在 Web Worker 的 `blob:` 上下文中解析 URL，`blob:` 上下文没有基础 URL，相对路径会触发 `Failed to parse URL` 错误导致所有 tile 静默失败。

## 6. 样式切换机制（`setStyle` + `diff: false`）

MapLibre GL v4 中，`setStyle()` 的默认行为是 `{ diff: true }`，此模式下：
- 只触发 `styledata` 事件
- **不触发** `style.load` 事件

而自定义图层的重新加载依赖 `style.load` 事件，因此必须显式传入 `{ diff: false }`：

```typescript
map.setStyle(url, { diff: false });   // ✅ 触发 style.load
map.once("style.load", () => reinitLayers(map));
```

若使用默认 `diff: true`，切换底图后飞机图标和所有自定义图层都会消失。

## 7. `reinitLayers` 图层重建流程

每次底图样式切换后（包括初始加载、`switchStyle`、`switchProvider`、错误回退），`reinitLayers` 被调用以重新添加所有自定义内容：

```
reinitLayers(mapInstance)
│
├── applyBilingualLabels()          # 修改底图文字为双语（中英文）
│
├── 添加 terrain-dem source         # 高程瓦片数据源（terrarium 编码）
├── 添加 terrain-hillshade 图层      # 山地阴影（hillshade 类型，intensity=0.35）
│
├── loadMapIcon("icon-plane", ...)  # 加载飞行中图标（SVG → canvas）
├── loadMapIcon("icon-plane-ground", ...) # 加载地面图标
│
└── addCustomLayers(mapInstance)    # 添加所有业务数据图层
    │
    ├── weather-hubs source (GeoJSON)
    │   ├── temp-circles layer
    │   ├── wind-arrows layer
    │   └── wind-speed-labels layer
    │
    ├── airports source (GeoJSON)
    │   ├── airport-points layer
    │   └── airport-labels layer
    │
    ├── airports-highlight source (GeoJSON)
    │   ├── airport-highlight-points layer
    │   └── airport-highlight-labels layer
    │
    ├── aqi-hubs source (GeoJSON)
    │   ├── aqi-heatmap layer
    │   ├── aqi-circles layer
    │   └── aqi-labels layer
    │
    └── flights source (GeoJSON)
        ├── flight-density layer       # 密度热力图
        ├── selected-flight-point layer # 选中高亮圆
        ├── planned-route-line layer   # 计划航路线
        ├── selected-flight-track-line layer # 历史轨迹线
        ├── selected-flight-track-arrows layer # 轨迹方向箭头
        └── flight-points layer        # 飞机图标 symbol
```

## 8. 飞机图标配置

### 图标文件

| 文件 | 图标 ID | 颜色 | 用途 |
|---|---|---|---|
| `src/icons/plane.svg` | `icon-plane` | `#60a5fa`（蓝色） | 飞行中（altitude > 100 ft） |
| `src/icons/plane_ground.svg` | `icon-plane-ground` | `#94a3b8`（灰色） | 地面停放 |

图标以 SVG 文件形式存储，通过 `loadMapIcon()` 函数在运行时渲染到 canvas 并调用 `map.addImage()` 注册到 MapLibre。

> **注意：** 图标加载时**不使用** `{ sdf: true }` 选项。SDF（Signed Distance Field）模式会将图标视为单色蒙版并用 `icon-color` paint 属性着色；由于 SVG 本身已包含颜色，启用 SDF 会干扰颜色渲染。如需运行时变色，请在 SVG 中使用纯色填充并启用 SDF，同时在 layout 中配置 `icon-color`。

### Symbol 图层配置

`flight-points` 图层的关键 layout 属性：

```typescript
layout: {
    // 根据高度选择图标：>100ft 用飞行图标，否则用地面图标
    "icon-image": [
        "case",
        [">", ["coalesce", ["get", "altitude_ft"], 0], 100],
        "icon-plane",
        "icon-plane-ground"
    ],
    // 图标大小随缩放级别线性插值
    "icon-size": [
        "interpolate", ["linear"], ["zoom"],
        3, 0.06,   // 全国视角：很小
        6, 0.10,   // 省级视角
        10, 0.15,
        14, 0.20   // 城市视角：较大
    ],
    // 图标旋转角度使用 heading 属性
    "icon-rotate": ["coalesce", ["get", "heading"], 0],
    // 与地图旋转同步（飞机始终指向真实方向）
    "icon-rotation-alignment": "map",
    "icon-allow-overlap": true,     // 允许图标重叠
    "icon-ignore-placement": true,  // 忽略标签碰撞检测
}
```

### 自定义飞机图标颜色

修改 `src/icons/plane.svg` 中的 `fill` 属性即可更改飞行中图标颜色，修改 `plane_ground.svg` 更改地面图标颜色。

**深色底图推荐配色：**
- 飞行中：`#facc15`（亮黄）或 `#22d3ee`（青色）—— 与深色底图对比鲜明
- 地面：`#94a3b8`（灰色）—— 区分于飞行状态

## 9. 地形阴影图层

`terrain-hillshade` 使用 Amazon 公共高程数据（terrarium 编码），始终在底图符号层之下渲染：

```typescript
{
    id: "terrain-hillshade",
    type: "hillshade",
    source: "terrain-dem",
    paint: {
        "hillshade-intensity": 0.35,         // 阴影强度（0-1）
        "hillshade-shadow-color": "#5a4630", // 阴影颜色（棕色）
        "hillshade-highlight-color": "#ffffff",
        "hillshade-accent-color": "#5a4630",
    }
}
```

如需关闭或调整阴影强度，修改 `hillshade-intensity` 值（0 为关闭）。

## 10. 双语地名标注（`applyBilingualLabels`）

每次样式切换后自动执行，将底图的文字图层从英文改为中英文双语显示，方法是遍历所有 `type === "symbol"` 的图层并修改 `text-field` 属性为：

```
{name:zh} {name:en}
```

## 11. 自动回退与错误处理

### 初始化回退（`determineInitialStyle`）

1. 读取 `VITE_MAP_DEFAULT_PROVIDER` 配置
2. 若 provider 为 `maptiler`，HTTP 探测样式 URL 是否可访问
3. 若探测失败，尝试 Stadia（若有 Key）
4. 最终回退到 OpenFreeMap（无需 Key）
5. 回退时设置 `mapFallbackActive = true`，触发顶部提示条

### 运行时回退（`mapInstance.on("error")`）

监听 MapLibre 运行时错误：
- HTTP 403（密钥无效）→ 切换到 Stadia 并重建图层
- HTTP 429（配额用尽）→ 切换到 Stadia 并重建图层

## 12. 自定义修改指南

### 修改默认样式

在 `.env` 中修改 `VITE_MAP_DEFAULT_STYLE`，重启开发服务器即生效：

```dotenv
VITE_MAP_DEFAULT_STYLE=outdoor-v2
```

### 新增 MapTiler 样式

在 `MapView.vue` 的 `MAPTILER_STYLES` 数组中追加：

```typescript
const MAPTILER_STYLES = [
    ...
    { id: "topo-v2", label: "地形地貌" },  // 新增
];
```

MapTiler 完整样式列表参考：https://cloud.maptiler.com/maps/

### 新增 Stadia 样式

在 `STADIA_STYLES` 数组中追加，样式 ID 参考 Stadia 文档：https://docs.stadiamaps.com/map-styles/

### 修改初始视口

在 `.env` 中修改：

```dotenv
VITE_MAP_INIT_LNG=121.4737   # 上海
VITE_MAP_INIT_LAT=31.2304
VITE_MAP_INIT_ZOOM=8
```

### 修改地形阴影参数

在 `reinitLayers()` 函数中找到 `terrain-hillshade` 图层定义，修改 paint 属性。

### 修改飞机图标大小

在 `addCustomLayers()` 中找到 `flight-points` 图层的 `icon-size` 插值表达式，调整各缩放级别对应的大小系数。

### 禁用地形阴影

在 `reinitLayers()` 中删除或注释 `terrain-dem` source 和 `terrain-hillshade` layer 的添加代码。

```

├── MapLibre GL JS
│   ├── Web Worker (blob: URL)

    └── https://tiles.openfreemap.org/...
```

## 14. 关键注意事项

1. **`diff: false` 必须传**：`map.setStyle(url, { diff: false })` 是触发 `style.load` 事件的必要条件（MapLibre v4 默认 `diff: true` 只触发 `styledata`）。
2. **`transformRequest` 必须返回绝对 URL**：Web Worker 在 `blob:` 上下文无法解析相对路径。
3. **`markRaw(mapInstance)`**：MapLibre Map 实例必须用 `markRaw()` 包裹后赋给 Vue ref，防止 Vue 的响应式代理包装导致内部方法调用异常。
4. **图标 SDF 模式**：当前飞机图标不启用 SDF，颜色写死在 SVG 中；若需动态着色请先启用 `{ sdf: true }` 并移除 SVG 内颜色定义。
5. **地图样式切换 vs Provider 切换**：样式切换（`switchStyle`）在同一 provider 内切换；Provider 切换（`switchProvider`）切换到不同数据源，两者均调用 `reinitLayers` 重建自定义图层。
