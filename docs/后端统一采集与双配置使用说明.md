# 后端统一采集与双配置使用说明

## 1. 目标

本说明用于定义后端的统一工作模式：

- **统一采集**：后端定时采集实时层、环境层、商业层数据。
- **统一缓存**：后端在进程内缓存最新快照。
- **统一下发**：前端只调用后端接口，不直接调用第三方服务。

---

## 2. 三层数据采集架构

### 2.1 实时层（OpenSky Network）

- **数据内容**：全球在线飞机位置、速度、高度、航向、飞机类别（extended=1）
- **采集范围**：`OPENSKY_BBOX` 留空 = 全球（约 6000 架）；也可指定经纬度矩形
- **轮询策略（开发档）**：全时段匀速 90 秒，约 960 次/日（3840 点/日，预算 4000 点/日）
- **容错**：OpenSky 失败时自动回退 mock 数据，不中断服务
- **注册地址**：https://opensky-network.org/

### 2.2 环境层（OpenWeather）

- **数据内容**：20 个全球航空枢纽的实时天气（温度、风速、云量、降雨）+ 空气质量（AQI、PM2.5/PM10/O₃/NO₂/CO）
- **采集策略**：asyncio.Semaphore(5) 并发拉取，单枢纽超时不影响整批
- **默认 20 枢纽**：CAN/SZX/PEK/PVG/HKG/NRT/ICN/SIN/DXB/DOH/LHR/CDG/FRA/AMS/JFK/LAX/ORD/ATL/SYD/GRU
- **自定义枢纽**：在 `.env` 中设置 `OPENWEATHER_HUBS=CAN,HKG,SIN`（逗号分隔 IATA）
- **刷新间隔**：300 秒（5 分钟），约 40 次/轮次，远低于 Developer Plan 上限（3000 次/分钟）
- **注册地址**：https://openweathermap.org/api

### 2.3 商业层（AirLabs）

- **数据内容**：航班号、起降机场 IATA、航空公司、机型、飞行状态
- **采集策略**：每日一次 `/flights` 批量拉取，一次 API 调用覆盖全部航班（~5800 条）
- **额度消耗**：约 30 次/月，Free 计划 1000 次/月剩余 970 次
- **富集方式**：按 `hex`（ICAO24）与实时层数据合并，自动写入 FlightDetail
- **注册地址**：https://airlabs.co/

## 4. API Key 申请与填写

### 4.1 申请地址汇总

| 服务        | 申请地址                       | 免费额度                                      |
| ----------- | ------------------------------ | --------------------------------------------- |
| OpenSky     | https://opensky-network.org/   | 4000 点/日（注册用户）                        |
| OpenWeather | https://openweathermap.org/api | 60 次/分钟（Free），3000 次/分钟（Developer） |
| AirLabs     | https://airlabs.co/            | 1000 次/月                                    |

### 4.2 填写步骤

```powershell
# 1. 从模板复制
cp server/.env.development.example server/.env

# 2. 用编辑器打开 server/.env，将以下三处占位符替换为真实值：
OPENSKY_USERNAME="your-opensky-username"     → 填写 OpenSky 注册用户名
OPENSKY_PASSWORD="your-opensky-password"     → 填写 OpenSky 密码
OPENWEATHER_API_KEY="your-openweather-api-key"  → 填写 OpenWeather API Key
AIRLABS_API_KEY="your-airlabs-api-key"      → 填写 AirLabs API Key
```

> **重要**：`.env` 已在 `.gitignore` 中排除，不会被提交到仓库。  
> 请勿在 `.env.development.example` 或任何可提交文件中填入真实 Key。

### 4.3 各字段不填写时的行为

| 字段                        | 不填写时的影响                                                |
| --------------------------- | ------------------------------------------------------------- |
| `OPENSKY_USERNAME/PASSWORD` | 以匿名用户访问（额度降至 400 点/日），全球模式下约 1 小时耗尽 |
| `OPENWEATHER_API_KEY`       | 环境层采集失败，天气/AQI 缓存为空，前端显示无天气数据         |
| `AIRLABS_API_KEY`           | 商业层跳过，航班无航班号/起降机场/机型等富集字段              |

## 5. 统一下发接口（完整列表）

### 5.1 航班接口

| 端点                                    | 说明                                                                                 |
| --------------------------------------- | ------------------------------------------------------------------------------------ |
| `GET /api/v1/flights`                   | 航班列表，支持 `callsign`/`bbox`/`page`/`page_size` 筛选                             |
| `GET /api/v1/flights/{flight_id}`       | 单航班详情（含起降机场、机型、状态），额外返回 `departure_weather`/`arrival_weather` |
| `GET /api/v1/flights/{flight_id}/track` | 航迹点列表，支持 `start`/`end` 时间段参数                                            |
| `GET /api/v1/flights/summary/stats`     | 聚合统计：总架次、在地/空中分布、分类/高度/速度段分布、前20航司前缀                  |

### 5.2 数据中心接口

| 端点                                            | 说明                                                              |
| ----------------------------------------------- | ----------------------------------------------------------------- |
| `GET /api/v1/datahub/status`                    | 三层采集状态（source/last_success/last_error/count）              |
| `GET /api/v1/datahub/weather`                   | 环境层天气缓存，返回 `{"CAN":{...},"HKG":{...},...}` IATA 键字典  |
| `GET /api/v1/datahub/weather/nearest?lat=&lon=` | 按经纬度返回最近且有缓存数据的枢纽天气 + AQI，不发起外部请求      |
| `GET /api/v1/datahub/air_quality`               | 空气质量缓存，结构同 weather，含 AQI 和 8 种污染物浓度            |
| `GET /api/v1/datahub/commercial`                | 商业层缓存（AirLabs 富集数据）                                    |
| `GET /api/v1/datahub/quota`                     | 各 API 今日调用计数与预算（opensky/openweather/airlabs/playback） |
| `GET /api/v1/datahub/snapshot`                  | 聚合快照：`status + flights + weather + air_quality + commercial` |

### 5.3 历史回放接口

| 端点                                         | 说明                                                                            |
| -------------------------------------------- | ------------------------------------------------------------------------------- |
| `GET /api/v1/playback?start=&end=&interval=` | 按时间范围查询历史航班快照帧；`interval` 单位秒（30~3600），范围最大 48h/2000帧 |

**返回结构示例：**
```json
{
  "start": "2026-04-24T09:00:00+00:00",
  "end":   "2026-04-24T10:00:00+00:00",
  "interval_seconds": 300,
  "total_frames": 12,
  "frames": [
    { "ts": "2026-04-24T09:02:13+00:00", "flights": [{"id":"icao24-abc","lat":23.4,"lon":113.3,...}] },
    ...
  ]
}
```

快照存储配置（`.env` 可覆盖）：
- `PLAYBACK_SNAPSHOT_INTERVAL_SECONDS=300`（每 5 分钟存一帧，约 288 帧/天）
- `PLAYBACK_TTL_HOURS=24`（保留最近 24 小时，自动清理）

### 5.4 其他

| 端点                           | 说明                                       |
| ------------------------------ | ------------------------------------------ |
| `GET /api/v1/health`           | 健康检查，返回 `{"status":"ok","db":"ok"}` |
| `WS  /api/v1/ws/flights`       | WebSocket 实时推送（事件驱动 + 20 秒心跳） |
| `GET /debug/flights-dashboard` | 后端调试可视化页面                         |

## 7. 日志说明

后端日志格式：`时间 | 级别 | 模块 | 消息`

常见日志含义：

| 日志内容                                                                                    | 含义                                                     |
| ------------------------------------------------------------------------------------------- | -------------------------------------------------------- |
| `unified pipeline started with profile=development realtime=90s env=300s commercial=86400s` | 服务正常启动，显示各层间隔                               |
| `AirLabs bulk snapshot: N flights enriched`                                                 | 商业层采集成功，N 条航班已富集                           |
| `environment collected: N/20 hubs OK`                                                       | 环境层采集完成，N 个枢纽成功（<20 表示部分超时，属正常） |
| `Weather fetch failed for XXX: [TimeoutError]`                                              | 单枢纽天气请求超时，已跳过，不影响其他枢纽               |
| `Retrying N failed hub(s) after 5 s: [...]`                                                 | 首次 gather 后对失败枢纽自动重试一次，大多数情况下可恢复 |
| `environment collection failed [TimeoutError]: TimeoutError`                                | 整个环境层本轮失败（极少见），下轮自动重试               |
| `realtime collection failed [...]`                                                          | OpenSky 请求失败，将回退 mock 数据                       |

## 8. 密钥安全

- `.env` 已在根 `.gitignore` 中排除，**永远不会**被提交到仓库。
- `.env.development.example` 只包含占位符，可安全提交。
- 若密钥曾在截图、聊天或提交历史中暴露，请立即在各供应商后台轮换（Revoke + 新建）。

## 9. 验证记录（2026-04-24）

- OpenSky 全球模式：~7000 架飞机，90s 匀速轮询，0 失败
- AirLabs 批量富集：~7200 条，每日 1 次
- OpenWeather 多枢纽：初次 gather 后自动重试失败枢纽，最终 17~20/20 成功（国内网络到 OpenWeather API 存在抖动，属正常）
- `/api/v1/datahub/air_quality`：返回 IATA 键字典，含 AQI + 8 种污染物
- `/api/v1/datahub/weather/nearest`：按坐标返回最近有缓存枢纽，不返回空数据
- `/api/v1/datahub/quota`：含 opensky/openweather/airlabs 今日调用量 + playback 配置
- `/api/v1/flights/summary/stats`：含 total/by_category/by_altitude_band/by_speed_band/top_callsign_prefixes
- `/api/v1/playback`：启动 5 分钟后即可返回有效帧，快照每 300s 一帧，TTL 24h
- `/api/v1/flights/{id}`：响应中含 `departure_weather`/`arrival_weather` 字段（枢纽机场有数据，非枢纽为 null）
