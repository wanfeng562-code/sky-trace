# 后端统一采集与双配置使用说明

## 1. 目标

本说明用于定义后端的统一工作模式：

- 统一采集：后端定时采集实时层、环境层、商业层数据。
- 统一缓存：后端在进程内缓存最新快照。
- 统一下发：前端只调用后端接口，不直接调用第三方服务。

## 2. 两套配置

后端支持两套运行档位，通过 APP_PROFILE 控制：

- development：开发配置（默认）
- release：发行配置

示例模板文件：

- server/.env.development.example
- server/.env.release.example

默认策略：

- 实时层主数据源：OpenSky
- 实时层容错：OpenSky失败时回退到mock
- 商业层刷新周期：每日一次

## 3. 推荐频率

development（默认）：

- 实时层（OpenSky）：
  - 活跃时段：15秒
  - 非活跃时段：40秒
  - 活跃时段起点：08:00
  - 按 4000 次/日预算自动计算活跃时长，当前推导结束时间约为 20:16
- 环境层：300秒
- 商业层：86400秒（每天一次）

计算说明：

- 设活跃时长为 $x$ 秒，则每日调用量为：
  $$\frac{x}{15} + \frac{86400-x}{40} = 4000$$
- 解得 $x=44160$ 秒，即 12 小时 16 分钟。
- 从 08:00 起算，活跃窗口结束于约 20:16。

release：

- 实时层：10秒
- 环境层：120秒
- 商业层：86400秒（每天一次）

## 4. 启动方式

1. 在 server 目录准备 .env（可由 development 模板复制）。
2. 设置 APP_PROFILE=development 或 APP_PROFILE=release。
3. 启动后端：

```powershell
cd server
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

## 5. 统一下发接口

- GET /api/v1/flights
- GET /api/v1/flights/{flight_id}
- GET /api/v1/flights/{flight_id}/track
- GET /api/v1/flights/summary

新增统一聚合接口：

- GET /api/v1/datahub/status
  - 查看采集状态、配置档位、三层刷新间隔、最近错误。
- GET /api/v1/datahub/weather
  - 获取后端缓存的环境层数据（OpenWeather）。
- GET /api/v1/datahub/commercial
  - 获取后端缓存的商业层数据（AeroDataBox/Aviationstack）。
- GET /api/v1/datahub/snapshot
  - 一次返回 status + flights + weather + commercial。

## 6. 与额度限制的关系

- 前端不直接访问第三方，避免页面刷新导致额度激增。
- 商业接口按天刷新，满足额度受限场景。
- 实时显示依赖 OpenSky 实时层和后端缓存，支持高频可视化。

## 7. 密钥安全

- .env 文件应只放在本地并加入忽略。
- 若密钥曾在截图、聊天或提交中暴露，请立即在供应商后台轮换。
