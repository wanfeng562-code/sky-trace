# Sky-Trace 环境与配置说明

> 由模板升级为**当前可用配置清单**（2026-05-18）。完整示例见 `server/.env.development.example`、`client/.env.development`。

## 1. 环境基线

| 项 | 要求 |
|----|------|
| 操作系统 | Windows 10/11 / Linux（云服务器） |
| Python | 3.10+（建议 3.11） |
| Node.js | 20 LTS+ |
| npm | 10+ |

## 2. 目录约定

| 目录 | 说明 |
|------|------|
| `client/` | Vue 3 前端 |
| `server/` | FastAPI 后端 |
| `desktop/` | PySide6 壳（可选） |
| `docs/` | 文档 |
| `server/data/` | SQLite（已在 `.gitignore`） |

## 3. 后端关键环境变量（`server/.env`）

| 变量 | 说明 | 默认/备注 |
|------|------|-----------|
| `APP_PROFILE` | `development` / `release` | 控制采集间隔 |
| `APP_HOST` | 监听地址 | 云部署用 `0.0.0.0` |
| `APP_PORT` | 端口 | `8000` |
| `OPENSKY_CLIENT_ID` / `SECRET` | OpenSky OAuth2 | 不填则匿名 400 点/日 |
| `OPENWEATHER_API_KEY` | 天气 + AQI | 必填才有环境层 |
| `AIRLABS_API_KEY` | 商业富集 | 可选 |
| `FR24_PROXY_URL` | FR24 Worker 代理 | 可选，`/?url=` 结尾 |
| `MAPTILER_API_KEY` / `STADIA_API_KEY` | 瓦片代理 | 可选 |
| `SQLITE_PATH` | 数据库路径 | `./data/sky_trace.db` |
| `CORS_ALLOW_ORIGINS` | 前端源 | 含 `https://sky-trace.pages.dev` |
| `HTTP_PROXY` | 出站代理 | 国内网络可选 |
| `PLAYBACK_SNAPSHOT_INTERVAL_SECONDS` | 回放快照间隔 | `300` |
| `PLAYBACK_TTL_HOURS` | 回放保留 | `24` |
| `TRACKS_PERSIST_INTERVAL_SECONDS` | 轨迹落库节流 | `60` |
| `TRACKS_TTL_HOURS` | 轨迹保留 | `24` |
| `WS_BROADCAST_MIN_INTERVAL_SECONDS` | WS 全量广播节流 | `2.0` |
| `ENVIRONMENT_GRID_MAX_CELLS` | 每轮 GRD 刷新上限 | `40` |

## 4. 前端环境变量（`client/.env.development`）

| 变量 | 说明 |
|------|------|
| `VITE_API_BASE_URL` | 如 `http://127.0.0.1:8000/api/v1` |
| `VITE_WS_URL` | 如 `ws://127.0.0.1:8000/api/v1/ws/flights` |
| `VITE_MAPTILER_KEY` / `VITE_STADIA_KEY` | 底图 Key（可选） |
| `VITE_MAP_DEFAULT_PROVIDER` | `maptiler` / `stadia` / `openfreemap` |
| `VITE_MAP_DEFAULT_STYLE` | 如 `streets-v2-dark` |
| `VITE_MAP_INIT_LNG` / `LAT` / `ZOOM` | 初始地图视野 |

生产构建时 `VITE_API_BASE_URL` / `VITE_WS_URL` 须指向**公网后端**。

## 5. 安全约定

- `.env`、`server/data/*.db` 不得提交。
- 仅 `.env.*.example` 可含占位符。
- Key 泄露后须在供应商后台轮换。
