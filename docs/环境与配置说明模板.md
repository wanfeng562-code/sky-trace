## 2. 环境基线
- 操作系统：Windows 10/11（开发主环境）
- Python：3.10+
- Node.js：20 LTS+
- npm：10+
- Qt for Python：PySide6 6.7+
- Git：2.40+

## 3. 项目目录约定
- client：前端工程（Vue3）
- server：后端工程（FastAPI）
- desktop：桌面壳工程（PySide6）
- docs：项目文档

## 4. 环境变量文件约定
- server/.env.example：后端示例配置（可提交）
- server/.env：后端本地配置（不可提交）
- client/.env.development：前端开发配置（可提交非敏感项）
- client/.env.local：前端本地配置（不可提交）

## 5. 后端配置模板（server/.env.example）

```env
APP_NAME=Sky-Trace-Server
APP_ENV=development
APP_HOST=127.0.0.1
APP_PORT=8000
LOG_LEVEL=INFO

# 数据采集
DATA_SOURCE_MODE=mock
FLIGHT_API_BASE_URL=https://example.com/api
FLIGHT_API_KEY=replace_me
FETCH_INTERVAL_SECONDS=5
REQUEST_TIMEOUT_SECONDS=10

# 存储
SQLITE_PATH=./data/sky_trace.db
CACHE_TTL_SECONDS=30

# CORS
CORS_ALLOW_ORIGINS=http://localhost:5173

# WebSocket
WS_HEARTBEAT_SECONDS=20
WS_MAX_CLIENTS=50
```

## 6. 前端配置模板（client/.env.development）

```env
VITE_APP_NAME=Sky-Trace
VITE_API_BASE_URL=http://127.0.0.1:8000/api/v1
VITE_WS_URL=ws://127.0.0.1:8000/api/v1/ws/flights
VITE_MAP_PROVIDER=maplibre
VITE_ENABLE_MOCK=false
# MapTiler Cloud API Key（主底图，https://cloud.maptiler.com/account/keys/）
VITE_MAPTILER_KEY=
# Stadia Maps API Key（备用底图，https://client.stadiamaps.com/）
VITE_STADIA_KEY=
```

## 7. 启动步骤模板

### 7.1 后端启动（Windows）
1. 进入 server 目录
2. 创建并激活虚拟环境
3. 安装依赖 requirements.txt
4. 拷贝 .env.example 为 .env 并修改配置
5. 启动服务：uvicorn app.main:app --reload --host 127.0.0.1 --port 8000

### 7.2 前端启动
1. 进入 client 目录
2. 安装依赖：npm install
3. 启动开发服务：npm run dev
4. 默认访问地址：http://localhost:5173

### 7.3 Qt 桌面壳启动
1. 进入 desktop 目录
2. 安装 desktop 依赖
3. 启动桌面程序（命令按实际脚本填写）
4. 验证：Qt 页面可正常加载前端并访问后端

## 8. 配置项说明表（后端）

| 配置项                 | 默认值              | 是否敏感 | 说明           |
| ---------------------- | ------------------- | -------- | -------------- |
| APP_ENV                | development         | 否       | 运行环境       |
| APP_PORT               | 8000                | 否       | 服务端口       |
| FLIGHT_API_BASE_URL    | -                   | 否       | 外部数据源地址 |
| FLIGHT_API_KEY         | -                   | 是       | 外部数据源密钥 |
| FETCH_INTERVAL_SECONDS | 5                   | 否       | 抓取间隔       |
| SQLITE_PATH            | ./data/sky_trace.db | 否       | SQLite 路径    |

## 9. 配置项说明表（前端）

## 10. 密钥与安全规范
- 任何密钥、Token、账号密码不得提交到仓库。
- .env、.env.local 必须加入 .gitignore。
- 对外演示环境使用独立测试密钥。
- 失效密钥及时轮换并记录到变更日志。

## 11. 常见问题排查

### 11.1 端口冲突
- 症状：后端无法启动
- 处理：更换 APP_PORT 并同步更新 VITE_API_BASE_URL 与 VITE_WS_URL

### 11.2 跨域失败
- 症状：前端请求报 CORS 错误
- 处理：检查 CORS_ALLOW_ORIGINS 是否包含前端地址

### 11.3 WebSocket 无数据
- 症状：连接成功但无更新
- 处理：检查 DATA_SOURCE_MODE、抓取日志与推送循环
