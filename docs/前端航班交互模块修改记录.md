## [cite_start]1. 目标概述 [cite: 4]

[cite_start]本次在地图页完成「列表筛选 → 选班拉轨迹 + 拉详情」的闭环，并增加起降场天气展示与 WebSocket 断线重连、在线状态提示 [cite: 5]。实现上需统一三处：
1.  [cite_start]**数据形状一致性**: 类型与 GET 详情的数据形状（含 `last_position` 与顶层字段合并） [cite: 5]。
2.  [cite_start]**状态管理一致性**: Pinia 中选中、轨迹、详情的一致性与竞态控制 [cite: 5]。
3.  [cite_start]**入口统一**: 地图点选与列表行选共用同一选择入口，避免轨迹与详情错位 [cite: 5]。

[cite_start]此外，天气区域单独拆为 `weatherBlock.vue`，以避免在 Vite 默认的 runtime-only 构建下出现“有数据但不渲染”（仅显示 `<!>`）的问题 [cite: 6]。

## [cite_start]3. 分步实施 (对应需求 Step 1~7) [cite: 9]

### [cite_start]Step 1 · 类型定义 [cite: 10]
* [cite_start]**文件**: `types/flight.ts` [cite: 11]
* [cite_start]**内容**: 在既有 `FlightBrief` / `TrackPoint` / `ApiResponse` 基础上新增： [cite: 12]
    * [cite_start]`weatherInfo`: 温湿度、风、能见度、天气描述等 [cite: 13]。
    * [cite_start]`FlightDetail`: 扩展自 `FlightBrief`，包含机场、机型、状态、`departure_weather` / `arrival_weather` [cite: 14]。
    * [cite_start]`FlightQueryParams`: 列表查询参数（如 `callsign`、`bbox`、`page` 等） [cite: 15]。
    * [cite_start]`FlightStats`: 统计摘要 [cite: 16]。

### [cite_start]Step 2 · API 层 [cite: 17]
* [cite_start]**文件**: `services/api.ts` [cite: 18]
* [cite_start]**修改**: `fetchFlights` 增加可选入参，透传给 `GET /flights` [cite: 19]。
* [cite_start]**新增**: `fetchFlightDetail(flightId)`、`fetchFlightStats()` [cite: 20]。
* [cite_start]**联调关键**: 在 `fetchFlightDetail` 内将 `data.last_position` 中的运动学字段（`lat`、`lon` 等）合并到返回对象顶层，使组件可直接绑定 `FlightDetail` 而无需手动解包 [cite: 21, 22]。

### [cite_start]Step 3 · Pinia Store [cite: 23]
* [cite_start]**文件**: `stores/flight.ts` [cite: 24]
* [cite_start]**新增状态**: `searchKeyword`、`filterStatus` (all | airborne | on_ground)、`flightDetail`、`detailLoading`、`wsOnline` [cite: 25]。
* [cite_start]**计算属性**: `filteredFlights` (关键字 + 状态过滤，供列表与地图共用) [cite: 26]。
* [cite_start]**方法**: `loadFlightDetail`（包含竞态处理，仅写入当前选中的 ID） [cite: 27][cite_start]；`selectFlight(null)` 清空数据 [cite: 28]。
* [cite_start]**类型调整**: `Socket` 类型调整为可关闭句柄，确保在页面切换时正确释放连接 [cite: 29]。

### [cite_start]Step 4 · WebSocket 重连 [cite: 30]
* [cite_start]**文件**: `services/ws.ts` [cite: 31]
* [cite_start]**内容**: 实现退避重连机制（如 2s/4s/8s/15s/30s）；通过 `onStatusChange` 同步 `wsOnline` 状态；对外返回 `close()` 供销毁调用 [cite: 32]。

### [cite_start]Step 5 · 详情与天气组件 [cite: 33]
* [cite_start]**文件**: `components/FlightDetailCard.vue` (新建) - 展示航班基础信息、状态及起降天气 [cite: 34, 35]。
* [cite_start]**文件**: `components/weatherBlock.vue` (新建) - 以表格形式展示 `weatherInfo` 字段 [cite: 36, 37]。

### [cite_start]Step 6 · 列表面板 [cite: 39]
* **文件**: `components/FlightListPanel.vue`
* [cite_start]**内容**: 绑定搜索与筛选状态；集成基于 `wsOnline` 的连接异常提示及空列表提示 [cite: 40]。

### [cite_start]Step 7 · 地图页集成 [cite: 41]
* [cite_start]**文件**: `views/MapView.vue` [cite: 42]
* [cite_start]**内容**: 挂载 `FlightDetailCard`；统一 `handleSelectFlight` 方法，确保点选地图与列表行选均能同步加载轨迹与详情 [cite: 43, 44]。

## [cite_start]4. 联调说明 (接口、后端现象、环境) [cite: 45]

### [cite_start]4.1 与后端/数据的约定 [cite: 46]
* [cite_start]**字段对齐**: 详情中的天气对象字段需与前端 `weatherInfo` 定义（如 `temp_c`、`humidity_pct` 等）一致 [cite: 47]。
* [cite_start]**环境配置**: 通过 `VITE_API_BASE_URL` 指定 API 根路径 [cite: 48]。

### [cite_start]4.2 后端索引 [cite: 49]
* [cite_start]**示例路径**: `server/app/api/routes/flights.py`（负责组装 `departure_weather` / `arrival_weather`） [cite: 50][cite_start]。相关 `schemas` 与 `unified_pipeline` 以当前分支实际提交为准 [cite: 51]。

## [cite_start]5. 专项修复：天气有数据但页面不显示 [cite: 52]

* [cite_start]**原因**: Vite 默认的 runtime-only 构建下，在父组件中以字符串形式内联子组件会导致无法正确编译，DOM 仅出现 `` 注释 [cite: 53]。
* [cite_start]**处理**: 新增 `weatherBlock.vue` 单文件组件，并在 `FlightDetailCard` 中通过 `import` 正常引用 [cite: 54]。
